# I2V-CompBench 七维度重构方案

## 总体架构变更说明

### 核心范式转变

原设计：单张首帧图像输入 → 图内已有主体发生变化。

新设计：支持**单图输入**和**多图/多参考输入**两种模式。每个维度都可以包含两种输入模式，单图模式作为基础题型，多图组合模式作为进阶题型。

### 范式转变的依据

I2V 与 T2V 最大的不同在于：首帧图像已经锚定了主体、属性、位置、动作和背景。这意味着题目不能只停留在"图内已有主体发生变化"这个单一模式上。真实 I2V 使用场景中，用户经常提供多张参考图（人物照、服装照、场景照），要求模型将它们组合成视频。

因此，benchmark 应当覆盖两种能力：
1. **单图编辑控制**：对已有首帧内容做精确编辑
2. **多参考条件合成**：将多张参考图中的元素组合到同一视频场景中

### 新的七个维度总览

| 编号 | 维度名称 | 核心定义 |
|------|----------|----------|
| 1 | Attribute Binding | 主体属性/状态变化 + 多参考属性迁移 |
| 2 | Action Binding | 主体肢体动作的正确绑定 |
| 3 | Motion Binding | 主体轨迹运动（含单主体绝对方向 + 多主体相对位移） |
| 4 | Spatial Composition | 多参考主体的静态空间布局组合 |
| 5 | Background Dynamics | 环境状态变化 + 背景迁移/替换 |
| 6 | View Transformation | 镜头/视角控制 |
| 7 | Interaction Reasoning | 物体交互的因果推理与物理/社会常识 |

### Spatial 与 Motion 的边界重构

旧边界：
- Motion Binding = 单主体绝对方向位移（球向左移动）
- Spatial Relation = 多主体相对空间关系编辑（球从盒子左边移到右边）

新边界：
- **Motion Binding** = 所有涉及主体**轨迹位移**的任务，包含单主体绝对方向和多主体相对位移变化
- **Spatial Composition** = 将多个参考主体**放置到指定空间位置关系**中，测的是静态空间布局的正确性

判定规则：主体有没有在视频中发生位移？
- 有位移 → Motion Binding（不管是绝对方向还是相对关系变化）
- 没有位移，只是组合放置 → Spatial Composition

这个重构使 Spatial Composition 成为一个更独特的 I2V 维度——测"多参考物体的空间组装"；Motion Binding 外延扩大——覆盖所有物体位置位移场景。

### Generative Numeracy 不独立成维度

在多图输入范式下，输入几张参考图就天然定义了物体数量。数量正确性已经内嵌在 Spatial Composition 等维度的评测中。单图模式下的数量保持能力作为各维度评分框架中的通用评分项。

## 维度 1：Attribute Binding（属性绑定）

### 一、这个维度到底测什么

核心测试：主体属性或状态的变化能否被准确绑定到指定主体上。

单图模式：首帧图像中已有主体，prompt 指定其中某些主体发生属性或状态变化。测三件事：谁变、变什么、谁不该变。

多图模式：输入多张参考图（如一个人的照片、一件衣服的照片、一条裤子的照片），prompt 要求人物穿上参考图中的衣服和裤子。测的是属性迁移的绑定准确性。

### 二、两种输入模式的题型定义

**模式 A：单图属性变化（经典）**

输入：一张首帧图像 + 文本 prompt。
任务：首帧中已有主体发生属性变化。
示例：
- 首帧：白狗和棕狗并排站立
- Prompt："The white dog turns black while the brown dog remains unchanged."

**模式 B：多图属性迁移（新增）**

输入：多张参考图 + 文本 prompt。
任务：将参考图中的属性（服装、颜色、纹理等）迁移到目标主体上。
示例：
- 输入图1：一个站立的女性
- 输入图2：一件红色连衣裙
- 输入图3：一双白色运动鞋
- Prompt："The woman wears the red dress and white sneakers."

### 三、属性变化类型分层

**第一层：v1 主力（最推荐）**

颜色变化：white → blue, green → red, black → white
状态变化：empty → full, closed → open, dry → wet, dark → bright

优点：初态终态差异大，语言表达清楚，首帧容易合成，不容易语义冲突。

**第二层：可以做，但先少量**

外观附加件/局部细节：no hat → red hat, no glasses → glasses, clean → dirty, intact → broken
属性迁移：参考图中的服装 → 穿到主体上，参考图中的纹理 → 应用到物体上

**第三层：先谨慎**

材质/纹理：smooth → rough, plain → striped, wooden → metallic
首帧和终态都更难一眼看清，v1 不作为主力桶。

### 四、难度分桶

单图模式沿用 S×A 模型：

| 桶 | 定义 | 难度 |
|----|------|------|
| S1A1 | 单主体，单属性变化 | Easy |
| S1A2 | 单主体，双属性变化 | Easy-Medium |
| S2A1 | 双主体，单属性变化 | Medium |
| S2A2 | 双主体，双属性变化 | Medium-Hard |
| S3A1 | 三主体，单属性变化 | Hard |
| S3A2 | 三主体，双属性变化 | Hard |

多图模式额外增加难度轴——参考图数量与融合复杂度：

| 桶 | 定义 | 难度 |
|----|------|------|
| R2 | 2张参考图，单属性迁移 | Medium |
| R3 | 3张参考图，多属性迁移 | Hard |
| R3+ | 3张以上参考图，复合属性迁移 | Hard |

语义罕见度：每种模式中 80% 常见组合 + 20% 罕见组合。
- 常见组合：白狗变黑、杯子由空变满
- 罕见组合：蓝色苹果变金色、金属质感转木纹

### 五、首帧/参考图设计原则

**单图模式原则**

1. 首帧必须清楚给出初态：杯子明显是空的、门明显是关的、衬衫明显是白的。不要半开半关、半满半空。
2. 首帧必须服务于绑定：多主体场景中，主体必须可区分（位置、颜色、大小）。
3. 首帧不要提前泄露终态。
4. 多主体优先同类主体：两只狗、三个杯子，而不是一只狗加一辆车。

**多图模式原则**

1. 每张参考图只承载一个明确属性：一张图是一件衣服，不要一张图里同时塞衣服和裤子。
2. 参考属性必须视觉可区分：红色裙子和白色鞋子的颜色差异要大。
3. 目标主体的参考图必须给出清楚的身份信息。
4. 参考属性和目标主体之间不能有语义冲突：不要让参考图里的衣服尺寸和人物体型差异太大。

### 六、提示词限制规则

**结构建议**

单图模式三段式：`[主体锚定] + [变化要求] + [轻度动态背景]`
多图模式三段式：`[目标主体指代] + [参考属性描述] + [融合场景说明]`

**长度建议**

短 prompt：8 到 16 个英文词。最长不超过 25 个英文词。

**必须包含**

- 目标主体的明确指代
- 属性变化的明确描述（单图）或属性迁移的明确指令（多图）

**禁止包含**

- 强动作词：running wildly, jumping, spinning rapidly
- 镜头词：camera pans, cinematic orbit, dramatic zoom
- 风格修饰：watercolor style, anime style, cinematic lighting
- 绝对方向词：move left, move right（这属于 Motion Binding）

**运镜限制**

默认固定机位或极轻微推近/推远。不允许明显平移、摇摄、环绕、大幅 zoom。

### 七、对比组设计

每个属性变化类型应构造正反对照：
- 正：白狗变黑，棕狗不变
- 反：棕狗变白，白狗不变

### 八、示例题目模板

**单图模式**

基础题：
- 首帧：一个空杯子放在桌上
- Prompt：The empty cup slowly becomes full. The camera remains fixed.

多主体题：
- 首帧：两只气球，一红一蓝
- Prompt：The red balloon turns green while the blue balloon remains unchanged.

**多图模式**

基础属性迁移：
- 输入图1：站立的年轻女性
- 输入图2：蓝色牛仔外套
- Prompt：The woman puts on the blue denim jacket.

多属性迁移：
- 输入图1：坐着的男性
- 输入图2：条纹衬衫
- 输入图3：黑色礼帽
- Prompt：The man wears the striped shirt and the black hat.

### 九、最容易犯的错

1. 单图模式里把属性题出成动作题（"狗跑着变黑"→ 动作太强）
2. 多图模式里参考图信息过于模糊，模型无法明确识别目标属性
3. 多主体题里主体区分不够，导致绑定无法验证
4. 忽略了对比组设计，无法排除模型的偏向

## 维度 2：Action Binding（动作绑定）

### 一、这个维度到底测什么

核心测试：首帧中已有主体，prompt 指定谁做什么肢体动作，测动作能否正确绑定到指定主体，不串到别的主体身上。

单图模式：首帧中有多个主体，prompt 为不同主体分配不同动作。

多图模式：输入多个主体的参考照片（可选背景参考图），prompt 要求这些主体在指定场景中执行各自的动作。

### 二、两种输入模式的题型定义

**模式 A：单图动作分配（经典）**

输入：一张首帧图像 + 文本 prompt。
任务：首帧中已有主体执行指定肢体动作。
示例：
- 首帧：两个女人并排站在公园里
- Prompt："A single continuous medium-wide shot of two women. The woman on the left waves while the woman on the right claps. The camera remains fixed."

**模式 B：多图场景动作合成（新增）**

输入：多张主体参考图 + 可选背景参考图 + 文本 prompt。
任务：将参考图中的人物放入指定场景，并执行各自的动作。
示例：
- 输入图1：一个穿白衬衫的男性半身照
- 输入图2：一个穿黑外套的女性半身照
- 输入图3：一张咖啡厅内景照片
- Prompt："In the café, the man in white waves while the woman in black nods. The camera remains fixed."

### 三、动作类型与动词词表

**推荐的原子化动作词**

上肢：waves, claps, raises hand, points
头部：nods, turns head, tilts head, looks up
全身：bows, sits down, stands up, jumps
复合（仅限 Hard）：waves then bows

**不推荐的动作词**

语义松散：starts dancing, performs a gesture, moves energetically
交互性：picks up the cup, pushes the chair（这属于 Interaction Reasoning）
叙事性：greets someone, reacts happily

### 四、难度分桶（S×D 模型）

S = 需被动作约束的主体数，D = 动作区分难度（混淆强度）。

| 桶 | 定义 | 核心测试 |
|----|------|----------|
| S1D1 | 单主体单动作 | 校准桶，确认动作本身适不适合当题 |
| S2D1 | 双主体，一动一静 | 最基础 binding，动作不串到另一个主体 |
| S2D2 | 双主体双不同动作 | 双动作分配（wave + clap） |
| S2D3 | 双主体高混淆（同类主体、对称布局） | 高混淆条件下还能否绑对 |
| S3D2 | 三主体中等负载 | 三路动作分配（wave + clap + bow） |

多图模式额外增加：场景融合难度。

### 五、首帧/参考图设计原则

**单图模式**

1. 动作相关部位不遮挡、不出框。手、头、躯干等关键部位完整可见。
2. 优先上肢和头部动作（更容易在中景里同时装下多主体）。
3. 主体初始姿态为 neutral pre-action pose。
4. 多主体布局本身就是难度控制器：S2D3 用镜像布局 + 相似服装。

**多图模式**

1. 每张主体参考图提供清楚的人物身份信息（面部、服装）。
2. 背景参考图的空间结构要支持多人共存和动作执行。
3. 参考图中的人物不应已经在执行目标动作。

### 六、提示词限制规则

**结构建议**

四段式：`场景壳子 + 主体锚定 + 动作分配 + 运镜约束`

**动词必须原子化、视觉明确**

好：waves, claps, nods, bows, sits down
坏：starts dancing, performs a gesture, interacts with

**多主体 prompt 避免代词链**

坏："The woman on the left waves, and she then turns to her, while the other one claps."
好："The woman on the left waves while the woman on the right claps."

**运镜策略**

single continuous shot + camera remains fixed + no cuts。禁用：pan to follow, fast zoom, orbit around, cuts to another angle, handheld shaky camera。

### 七、对比组设计

动作分配的对比：
- 正：左边人挥手，右边人不动
- 反：右边人挥手，左边人不动

### 八、示例题目模板

**单图 S2D3 高混淆**

- 首帧：两个穿相似服装的年轻女性对称站立，一穿白衣一穿黑衣
- Prompt："A single continuous medium-wide shot of two similarly dressed women. The woman on the left in the white shirt waves while the woman on the right in the black shirt claps. The camera remains fixed."

**多图场景合成**

- 输入图1：男性A半身照
- 输入图2：男性B半身照
- 输入图3：公园长椅场景照
- Prompt："In the park, the man on the left waves while the man on the right sits down on the bench. The camera remains fixed."

### 九、最容易犯的错

1. 把动作题出成运动题："the man runs to the left" → 更像 Motion Binding
2. 把动作题出成交互题："the man picks up the cup" → 更像 Interaction Reasoning
3. 首帧已经暗示动作完成（坐姿出 sit down）
4. 运镜太强，不知道在测动作还是镜头调度
5. 多图模式里人物身份信息不够，模型无法对应

## 维度 3：Motion Binding（运动绑定）

### 一、这个维度到底测什么

核心测试：主体在视频中发生的轨迹级运动是否正确。

这个维度在重构后外延扩大，包含两大类：

**类型 A：单主体绝对方向运动（经典）**

首帧中一个目标主体，prompt 指定它向六个 canonical directions 之一移动：left / right / up / down / toward camera / away from camera。

**类型 B：多主体相对位移变化（原 Spatial Relation 中的运动部分）**

首帧中多个主体，prompt 要求其中某个或某些主体发生位置位移，导致它们之间的相对空间关系发生改变。判定标准：只要主体在视频中发生了位移，就归这个维度。

**类型 C：多图组合运动（新增）**

输入多张参考图中的物体，prompt 要求某个物体在场景中做轨迹运动。

### 二、三种题型定义

**模式 A：单主体绝对方向运动**

输入：一张首帧图像 + 文本 prompt。
示例：
- 首帧：草地中央偏右有一只棕色小狗，左侧留有充足空间
- Prompt："The small brown dog moves to the left."

**模式 B：多主体相对位移**

输入：一张首帧图像 + 文本 prompt。
示例：
- 首帧：一个红球在蓝盒左边
- Prompt："The red ball moves to the right side of the blue box."

**模式 C：多图组合运动**

输入：多张参考图 + 文本 prompt。
示例：
- 输入图1：一座城市天际线的照片
- 输入图2：一架客机的照片
- Prompt："The airplane flies across the city skyline from left to right."

### 三、方向与运动类型

**平面方向（D1）**

left, right, up, down。最基础，靠 2D 平面位移表达。

**深度方向（D2）**

toward camera, away from camera。更难，需要首帧有明确透视和深度感。

**相对位移方向**

to the left of [B], above [B], in front of [B], between [B] and [C]。描述的是运动的终态与参照物的关系。

### 四、难度分桶

**类型 A（单主体绝对方向）沿用 S×D 模型**

| 桶 | 定义 | 特点 |
|----|------|------|
| S1-D1 | 简单场景 + 平面方向 | 单主体显著，背景干净 |
| S2-D1 | 中等场景 + 平面方向 | 有干扰物，目标靠属性唯一指代 |
| S2-D2 | 中等场景 + 深度方向 | 透视明确，toward/away |
| S3-D1 | 复杂场景 + 平面方向 | 小目标或边缘目标 |
| S3-D2 | 复杂场景 + 深度方向 | 最难桶 |

**类型 B（多主体相对位移）的分桶**

| 桶 | 定义 | 难度 |
|----|------|------|
| M2R1 | 2主体，1个位移关系变化，纯2D | Easy |
| M2R1-D | 2主体，1个位移关系变化，前后 | Medium |
| M3R1 | 3主体，1个核心位移变化 | Medium |
| M3R2 | 3主体，2个位移关系同时变化 | Hard |
| M4R2 | 4主体，多主体重排 | Hard |

数据配额建议：
- 类型 A 占 50%（100题），类型 B 占 35%（70题），类型 C 占 15%（30题）
- 方向平衡：平面方向 60%，深度方向 25%，多主体相对 15%

### 五、首帧/参考图设计原则

**通用原则**

1. 首帧必须为目标方向预留运动空间。向左题不能把目标放最左边；向前靠近镜头题目标不能已经特别大。
2. 目标主体边界清楚，与背景有足够对比。

**类型 A 特有原则**

1. 目标主体用外观属性唯一指代（the red balloon），不用关系指代（the dog on the left）。
2. toward/away 题首帧必须有明确深度线索：道路透视、地面消失点、前中后景层次。

**类型 B 特有原则**

1. 起始关系必须一眼可判。
2. 目标关系必须可达（留够空间）。
3. 主体必须可区分（颜色/大小/纹理）。
4. 前后关系必须有深度线索。

**类型 C 特有原则**

1. 每张参考图中的目标物体必须清楚可识别。
2. 场景参考图要有足够空间支撑运动轨迹。

### 六、提示词限制规则

**类型 A prompt 模板**

The [target] moves to the left/right/upward/downward/toward the camera/away from the camera.

target 只由类别名 + 颜色/纹理/尺寸/静态属性组成。

**类型 B prompt 要求**

- 必须包含：目标主体 + 参照主体 + 目标关系状态
- 用关系结果词：ends up to the right of, is repositioned above, moves into a position behind
- 禁止绝对方向词：move left, move right（这是类型A）

**通用禁止项**

- 局部动作词：wave, nod, wag, flap（属于 Action Binding）
- 语义动作词：run, walk, jump, dance（混入 Action）
- 多阶段词：first, then, after that
- 运镜词：pan, zoom, dolly, tracking shot
- 属性变化词：turn red, become larger（属于 Attribute Binding）
- 交互词：touch, collide, chase（属于 Interaction Reasoning）

**动词限制**

推荐中性运动动词：move, shift, drift。少用语义动作动词：walk, run, drive, fly。

### 七、对比组设计

必须构造方向对照：
- left / right 对照（相同物体，反方向）
- up / down 对照
- toward / away 对照
- 类型B中 A→B左边 / A→B右边 对照

### 八、示例题目模板

**类型 A 基础**

- 首帧：红色气球在画面中央偏下
- Prompt：The red balloon moves upward.

**类型 B pairwise**

- 首帧：白车在黑卡车后方
- Prompt：The white car is repositioned in front of the black truck.

**类型 B 三主体**

- 首帧：猫在狗左侧，椅子在右侧
- Prompt：The cat moves to the middle between the dog and the chair.

**类型 C 多图**

- 输入图1：城市天际线
- 输入图2：一架客机侧面照
- Prompt：The airplane flies across the city skyline from left to right.

### 九、最容易犯的错

1. 类型A和类型B混淆：类型A用外观指代目标，类型B用关系描述终态
2. 把运动题出成动作题："the man runs leftward" 混入了 Action
3. 把运动题出成镜头题：没有 camera instruction 却看起来像镜头在动
4. 首帧没给运动空间，导致题目不合理
5. 类型B中初始关系太模糊或目标关系不可达

---

## 维度 4：Spatial Composition（空间组合）

### 一、这个维度到底测什么

核心测试：多个参考主体/物体能否被正确放置到指定的空间位置关系中。

这个维度在重构后从"关系编辑"变为"空间组合"。它测的不再是"已有物体的位移"（那已归入 Motion Binding），而是"将多个来源不同的物体正确组装到一个场景中，并保持指定的空间关系"。

关键判定：主体在视频中不发生位移，题目关注的是**放置后的空间关系是否正确**。

### 二、题型定义

**模式 A：单图空间布局验证**

输入：一张包含多个物体的首帧图像 + 文本 prompt。
任务：prompt 描述场景中物体应当维持的空间关系，测试模型能否在视频中保持该布局。
示例：
- 首帧：猫在沙发上，绿植在沙发旁
- Prompt："The cat stays on the sofa and the plant remains beside the sofa. The camera remains fixed."

**模式 B：多图空间组装（核心题型）**

输入：多张独立物体/主体的参考图 + 可选场景参考图 + 文本 prompt。
任务：将参考图中的物体放置到场景中的指定空间位置。
示例：
- 输入图1：一张灰色沙发
- 输入图2：一只橘猫
- 输入图3：一盆绿植
- 输入图4（可选）：客厅空间照片
- Prompt："Place the cat on the sofa and the plant beside the sofa in the living room."

### 三、空间关系类型

v1 只保留六类基本关系：
- left of / right of（左右）
- above / below（上下）
- in front of / behind（前后）

可扩展关系：
- on（在……上面，接触性）
- beside / next to（在旁边）
- between（在中间）

### 四、难度分桶

| 桶 | 定义 | 难度 |
|----|------|------|
| C2-2D | 2个物体，纯2D关系（左右、上下） | Easy |
| C2-3D | 2个物体，3D关系（前后） | Medium |
| C3-2D | 3个物体，2D混合关系 | Medium |
| C3-3D | 3个物体，含3D关系 | Hard |
| C4+ | 4个及以上物体，复杂布局 | Hard |

额外难度轴——歧义度：
- 异类物体：猫 + 沙发 + 绿植（容易区分）
- 同类物体：三把不同颜色的椅子（需要属性区分）
- 同类近似物体：三把相似椅子（极高混淆）

### 五、参考图设计原则

1. 每张参考图只包含一个目标物体，背景尽量干净。
2. 物体的尺度、视角、光照要合理，不要参考图里的物体明显和场景不协调。
3. 场景参考图要有足够空间放置所有物体。
4. 前后关系的场景参考图必须有明确深度线索。
5. 不要在场景图中已经有和参考物体相似的物体，避免歧义。

### 六、提示词限制规则

**结构建议**

`[放置指令] + [物体A] + [空间关系] + [物体B] + [可选场景说明]`

示例：
- "Place the cat on the sofa and the plant beside the sofa in the living room."
- "Arrange the red chair to the left of the blue table, with the lamp behind the table."

**必须包含**

- 所有参考物体的指代
- 物体之间的明确空间关系

**禁止包含**

- 运动词：moves, shifts, repositions（有位移就归 Motion Binding）
- 动作词：waves, jumps
- 镜头词：camera pans, zoom
- 风格修饰

**运镜策略**

默认固定机位。可选轻微环绕以展示3D空间关系（仅限Hard桶少量使用）。

### 七、对比组设计

空间关系正反对照：
- 猫在沙发左边 / 猫在沙发右边
- 花瓶在桌子上面 / 花瓶在桌子下面
- 椅子在桌子前面 / 椅子在桌子后面

### 八、示例题目模板

**Easy（C2-2D）**

- 输入图1：红色花瓶
- 输入图2：蓝色书架
- 输入场景：客厅一角
- Prompt："Place the red vase to the left of the blue bookshelf."

**Medium（C3-2D）**

- 输入图1：橘猫
- 输入图2：灰色沙发
- 输入图3：一盆绿萝
- Prompt："Place the cat on the sofa and the plant beside the sofa."

**Hard（C4+）**

- 输入图1-4：四件不同家具
- 输入场景：一间空卧室
- Prompt："Arrange the bed against the back wall, the nightstand to the right of the bed, the desk on the left wall, and the chair in front of the desk."

### 九、最容易犯的错

1. 把空间组合题出成运动题：物体应该是被"放置"而不是"移动过去"
2. 参考图背景太杂，模型无法分离目标物体
3. 场景空间不够，放不下所有物体
4. 前后关系没有深度线索，天然歧义
5. 物体尺度和场景不匹配（参考图是特写，场景是全景）

## 维度 5：Background Dynamics（背景动态）

### 一、这个维度到底测什么

核心测试：对背景/环境区域施加变化，同时前景主体和其他未指定区域不应被破坏。

测三件事：
1. 模型能不能找对要变化的背景区域
2. 模型能不能只改环境，不把前景弄坏
3. 模型能不能把环境变化做得自然

### 二、两种题型

**类型 A：环境状态变化（经典）**

输入：一张首帧图像 + 文本 prompt。
任务：首帧中某个背景区域发生状态变化（天气、光照、纹理动态）。

子类型：
- 动态纹理型：云流动、海浪变大、树叶摇晃、烟雾扩散
- 场景状态型：天空变暗、远处起雾、开始下雨、灯光亮起

**类型 B：背景迁移/替换（新增）**

输入：主体所在图像 + 目标背景参考图 + 文本 prompt。
任务：将主体的背景替换为参考图中的背景。

示例：
- 输入图1：一个男人站在伦敦街头
- 输入图2：巴黎埃菲尔铁塔街景
- Prompt："The man is now walking on a Parisian street with the Eiffel Tower in the background."

### 三、难度分桶

**类型 A 三轴控制**

| 轴 | 低 | 中 | 高 |
|----|-----|-----|-----|
| 场景复杂度 | 开阔背景，前景少 | 有前景干扰 | 层次多、遮挡多 |
| 变化复杂度 | 单一变化 | 单区域强变化或两弱变化 | 多区域复合变化 |
| 语义推理负荷 | 直接可见 | 需区分局部与整图 | 有因果链 |

**类型 B 难度轴**

| 桶 | 定义 | 难度 |
|----|------|------|
| B1 | 简单主体 + 简单背景替换（相似场景类型） | Easy |
| B2 | 复杂主体 + 跨场景类型替换 | Medium |
| B3 | 多主体 + 复杂背景 + 需保持光照一致性 | Hard |

### 四、首帧/参考图设计原则

**类型 A**

1. 目标背景区域清楚可见（占画面15%-60%，按难度调）。
2. 必须有"非目标内容"作为 preserve 区。
3. 初态不能接近终态。

**类型 B**

1. 主体参考图中主体轮廓清楚，便于分离。
2. 背景参考图的空间结构要能容纳主体。
3. 两张参考图的光照条件差异要在合理范围内。
4. 背景替换不应改变主体的属性和动作。

### 五、提示词限制规则

**类型 A**

四部分：`目标背景区域 + 变化操作 + 时间展开 + 运镜约束`

v1 强制固定机位。禁止镜头运动词。

变化操作受控词表：drift, sway, ripple, spread, darken, brighten, become foggy, gradually, slowly。

可选 preserve clause（Medium/Hard）：while the person remains still。最多一条。

**类型 B**

三部分：`主体保持说明 + 目标背景描述 + 运镜约束`

示例：The man is now on a Parisian street with the Eiffel Tower visible in the background. The camera remains fixed.

### 六、对比组设计

类型A：
- 天空变暗 / 天空变亮（相反方向的状态变化）

类型B：
- 伦敦→巴黎 / 巴黎→伦敦（相反方向的背景迁移）

### 七、示例题目模板

**类型 A Easy**

- 首帧：一个人站在海边，天空晴朗
- Prompt："The clouds in the sky slowly drift while the person remains still. The camera remains fixed."

**类型 A Hard**

- 首帧：一辆车停在路边，远处有山景
- Prompt："Dark clouds gradually gather in the sky and the scene becomes dimmer. The car remains unchanged. The camera remains fixed."

**类型 B**

- 输入图1：一个女人站在伦敦街头的照片
- 输入图2：巴黎埃菲尔铁塔街景
- Prompt："The woman is now walking on a Parisian street with the Eiffel Tower in the background."

### 八、最容易犯的错

1. 类型A首帧里目标背景区域不够明确
2. 类型B背景替换后主体属性也跟着变了
3. 变化类型写得太抽象："the background changes dramatically"
4. 首帧本身已经接近终态
5. 类型B中光照不一致导致合成痕迹明显

## 维度 6：View Transformation（视角变换）

### 一、这个维度到底测什么

核心测试：联合设计首帧场景和镜头控制指令，测模型的镜头/视角控制能力。

本质："给你这张图，你接下来应该怎么拍它。"

这个维度的核心变量是 camera/view change，不是内容本身的语义编辑。

### 二、题型定义

以单图输入为主（多图输入对这个维度意义不大）。

输入：一张首帧图像 + 文本 prompt（包含镜头指令）。
任务：根据 prompt 中的镜头指令改变观察方式。

### 三、难度分桶（S×C 模型）

S = 场景复杂度，C = 运镜复杂度。

**场景复杂度**

| 级 | 特征 | 适合首帧 |
|----|------|----------|
| S1 | 单主体、背景刚性、无干扰 | 走廊里的人、桌上的花瓶 |
| S2 | 单/双主体、少量干扰、背景稍复杂 | 街边女人+少量路人 |
| S3 | 多主体、层次复杂、有遮挡 | 市场、车站（少量保留） |

**运镜复杂度**

| 级 | 特征 | 示例 |
|----|------|------|
| C1 | 单 camera command | slowly zoom in |
| C2 | 双约束（两个命令或一个命令+构图约束） | pan right + keep centered |
| C3 | 复合控制 | pan right + zoom in + centered（v1少做） |

推荐起步桶：S1C1, S2C1, S1C2, S2C2。

### 四、运镜词表

**v1 允许的核心运镜词**

基础运镜：zoom in, zoom out, pan left, pan right, tilt up, tilt down, keep the camera static
景别变化：push in, pull out, move to a closer view, move to a wider view
构图约束：keep the main subject roughly centered, maintain a centered framing

**v1 禁止的词**

cinematic, dramatic, orbit, crane shot, handheld, whip pan, POV, Dutch angle, shaky camera, over-the-shoulder

### 五、首帧设计原则

1. 主体清楚，背景可分层。
2. 背景以刚性为主。避免海浪、火焰、大面积树叶摆动、镜面强反射。
3. 初始构图与镜头命令匹配：zoom in 不能已经占满画面，wider view 不能已经太远。
4. 图文兼容性规则：
   - zoom in 适合：主体初始不太大、背景有层次
   - pan left/right 适合：场景横向有信息分布
   - tilt up/down 适合：场景纵向有信息
   - static camera 适合：主体有动作条件、背景刚性

### 六、提示词限制规则

硬约束：
- 每题必须显式出现至少一个 camera verb
- v1 每题最多两个镜头相关约束
- 构图约束最多一个
- 不允许用抽象描述代替具体镜头命令

允许：The camera slowly zooms in. / Keep the camera static. / Slowly pan right while keeping the woman centered.
不允许：Make the scene feel cinematic. / Use a dramatic close feeling.

### 七、对比组设计

- zoom in / zoom out 对照
- pan left / pan right 对照
- tilt up / tilt down 对照
- 运镜 / static camera 对照（相同场景）

### 八、示例题目模板

**S1C1**
- 首帧：走廊里的女人
- Prompt：The camera slowly zooms in.

**S2C1 with action**
- 首帧：草地上的狗
- Prompt：The dog runs across the field while the camera remains static.

**S1C2 双约束**
- 首帧：街边站立的女人
- Prompt：The camera slowly pans right while keeping the woman near the center of the frame.

### 九、最容易犯的错

1. 先写 prompt 再硬配图，导致不匹配
2. 把 Motion 题误生成为 Camera 题（"a man walks toward the viewer" 没有 camera instruction）
3. 图像场景不适合做镜头控制（背景全是动态元素）
4. 命令太多太花（v1 先把基础几何镜头控制做稳）
5. 图像和 prompt 难度不一致（S1 图像配 C3 复杂镜头）

## 维度 7：Interaction Reasoning（交互推理）

### 一、这个维度到底测什么

核心测试：多个物体/主体之间的交互行为是否符合物理常识和因果推理。

这个维度与 Action Binding 的关键区别：
- Action Binding = 单主体的肢体动作绑定（谁做什么动作）
- Interaction Reasoning = 多主体/物体之间的因果交互（A对B做了什么，B发生了什么变化）

它与 Motion Binding 的区别：
- Motion Binding = 轨迹位移控制（方向和路径）
- Interaction Reasoning = 交互过程中的因果链和物理状态变化

### 二、交互类型分类

**物理交互（Physical Interactions）**

涉及物理接触或力的传递，导致运动或状态变化。

子类型：
- 碰撞/推动：球撞倒积木、风吹落树叶
- 容器/承载：水倒入杯子、人坐到椅子上
- 抛投/接取：人扔球、猫扑蝴蝶
- 断裂/破坏：锤子砸碎玻璃、气球被扎破

**社会交互（Social Interactions）**

涉及有生命主体之间的社会行为。

子类型：
- 肢体交互：握手、拥抱、击掌
- 传递交互：递东西、接球
- 追逐/跟随：猫追老鼠、狗跟着主人

**功能交互（Functional Interactions）**

物体按其功能被使用。

子类型：
- 工具使用：用钥匙开门、用锤子钉钉子
- 装置操作：按下开关灯亮、打开水龙头水流出

### 三、难度分桶

| 桶 | 定义 | 难度 |
|----|------|------|
| I1 | 双物体，简单物理交互，因果链短 | Easy |
| I2 | 双物体，较复杂物理/社会交互 | Medium |
| I3 | 三物体或链式因果（A推B，B撞C） | Hard |
| I4 | 多物体，复合交互+状态变化 | Hard |

额外难度轴——因果推理深度：
- 浅因果：球碰杯子 → 杯子倒（直接因果）
- 中因果：风吹 → 树叶落 → 落在车上（两步因果）
- 深因果：人推多米诺骨牌 → 连锁倒下 → 最后一块撞铃响（多步因果）

### 四、首帧/参考图设计原则

**单图模式**

1. 交互双方都必须清晰可见，不能有严重遮挡。
2. 交互双方的空间距离要合理（不能隔太远也不能贴太近）。
3. 初态不能已经显示交互结果。
4. 场景要支持物理合理性（有地面、有重力感）。

**多图模式**

1. 每张参考图中的物体/主体清楚可识别。
2. 物体的尺度关系要合理（锤子不能比玻璃大十倍）。
3. 场景参考图要提供足够的交互空间。

### 五、提示词限制规则

**结构建议**

三段式：`[交互主体指代] + [交互行为描述] + [预期结果/状态变化]`

示例：
- "The ball rolls and hits the stack of blocks, causing them to fall over."
- "The cat jumps onto the table and knocks the vase off the edge."
- "Two people shake hands in the park."

**必须包含**

- 交互双方的明确指代
- 交互行为的明确描述
- 交互结果的明确或可推理的描述

**运镜策略**

默认固定机位或轻微跟踪。不允许强运镜分散对交互过程的注意力。

### 六、对比组设计

交互方向对照：
- A推B / B推A
- A递给B / B递给A

交互结果对照：
- 球碰杯子，杯子倒 / 球碰杯子，杯子不倒（力度不同）

### 七、示例题目模板

**Easy（I1 简单物理）**

单图模式：
- 首帧：一个球在斜面顶端，底部有一排积木
- Prompt："The ball rolls down the ramp and knocks over the blocks."

多图模式：
- 输入图1：一只猫
- 输入图2：一个鱼缸里有金鱼
- Prompt："The cat paws at the fishbowl, causing it to wobble."

**Medium（I2 社会交互）**

多图模式：
- 输入图1：男性A
- 输入图2：女性B
- 输入场景：公园长椅
- Prompt："The man hands a book to the woman on the park bench."

**Hard（I3 链式因果）**

单图模式：
- 首帧：桌上一排多米诺骨牌，最前面有一根手指
- Prompt："The finger pushes the first domino, triggering a chain reaction that topples all the dominoes in sequence."

### 八、最容易犯的错

1. 把交互题出成纯动作题：只写 "the cat jumps" 没有交互对象和结果
2. 交互结果不符合物理常识：轻轻碰一下就整个场景炸了
3. 因果链太长太复杂，无法在短视频中表达
4. 首帧已经显示交互结果（杯子已经倒了）
5. 交互双方在首帧中距离太远，交互不自然

## 附录：维度间的边界隔离规则

| 容易混淆的对 | 判定规则 |
|-------------|----------|
| Action vs Motion | 肢体动作 → Action；整体位移 → Motion |
| Action vs Interaction | 单主体自身动作 → Action；多主体因果交互 → Interaction |
| Motion vs Spatial | 有位移运动 → Motion；静态放置组合 → Spatial |
| Motion vs View | 主体在动 → Motion；镜头在动 → View |
| Attribute vs Action | 外观/状态变化 → Attribute；肢体动作 → Action |
| Background vs View | 环境自身变化 → Background；镜头运动 → View |
| Spatial vs Background | 可数主体空间布局 → Spatial；不可数环境区域 → Background |
| Interaction vs Motion | 有因果链和状态变化 → Interaction；纯位移无因果 → Motion |

## 附录：通用设计原则

### 对比组设计（所有维度通用）

每个维度中必须包含对比组样本，用于排除模型偏向。

### 语义罕见度（所有维度通用）

每个维度 200 题中，80%（160题）使用常见组合，20%（40题）使用罕见但合理的组合。

### 结构化元数据（所有维度通用）

每道题必须输出完整的结构化元数据，包含但不限于：
- dimension：维度名称
- input_mode：单图 / 多图
- input_images：输入图像列表及每张图的角色说明
- prompt：文本提示词
- difficulty_bucket：难度桶标签
- target_subjects：目标主体列表
- expected_change：预期变化类型
- preserve_constraints：保持约束
- semantic_rarity：common / uncommon
- contrastive_pair_id：对比组配对 ID

### 数据配额建议（所有维度通用）

每维度 200 题。

单图与多图的比例建议：
- Attribute Binding：60% 单图 + 40% 多图
- Action Binding：60% 单图 + 40% 多图
- Motion Binding：50% 类型A + 35% 类型B + 15% 类型C
- Spatial Composition：20% 单图 + 80% 多图（核心测多图空间组装）
- Background Dynamics：60% 类型A + 40% 类型B
- View Transformation：95% 单图 + 5% 多图
- Interaction Reasoning：50% 单图 + 50% 多图
