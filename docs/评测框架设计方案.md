## I2V-CompBench 评测框架设计方案

> 本文档基于「关于评测的思考.docx」中的 **Preserve–Transform 三轴架构** 和 **Exec 门控式评分** 核心思想，结合七维度重构方案中的维度定义与边界规则，综合 T2V-CompBench、T2I-CompBench++、VBench-I2V、UI2V-Bench 等相关工作的评测方法论，设计 I2V-CompBench 的完整评测框架。

---

## 一、评测哲学：Preserve–Transform 三轴

### 1.1 核心洞察

I2V 任务的本质是**有条件的选择性编辑**：给定输入图像 I、文本指令 P、生成视频 V={f₁,…,fₜ}。

输入图像定义初始组合状态：

```
C₀ = {E₀, A₀, R₀, S₀, B₀, K₀}
```

其中 E₀=实体集合、A₀=属性集合、R₀=关系集合、S₀=状态集合、B₀=背景/场景状态、K₀=相机/视角状态。

文本定义目标变化 ΔC —— 它不是重建整个世界，而是规定在 C₀ 上哪些元素要变、哪些不该变。

### 1.2 三轴定义

对每个样本，不管属于哪一类维度，都统一计算三轴：

| 轴 | 名称 | 定义 | 通俗表达 |
|----|------|------|----------|
| **E** | Exec (执行分) | 目标变化有没有执行对 | "该变的变对了吗？" |
| **P** | Preserve (保持分) | 未指定变化的部分有没有保持 | "不该变的没乱动吧？" |
| **C** | Coherence (连贯分) | 变化过程是否时序、几何、物理合理 | "变化过程自然吗？" |

形式化表达：

```
E = Exec(ΔC, V)
P = Preserve(C₀ \ ΔC, V)
C = Coherence(ΔC, V)
```

### 1.3 为什么是三轴而非双轴

1. **E 与 P 正交**：执行和保持是两种独立能力。E 高 P 低 = "能做但副作用大"；E 低 P 高 = "很保守做不出来"
2. **C 不可归入 E 或 P**：一个首尾碰巧正确但中间乱闪的视频，E 可能高但 C 该低；一个过程平滑但方向错的视频，C 高但 E 低
3. **论文写作优势**：三轴分别出雷达图，诊断精度远高于单一总分
4. **消融意义**：让研究者准确定位模型弱点在执行/保持/连贯哪个环节

### 1.4 各维度的三轴定义

| 维度 | E (执行) — 什么该变 | P (保持) — 什么不该变 | C (连贯) — 过程要求 |
|------|------|------|------|
| **Attribute Binding** | 指定主体的指定属性变化到位 | 非目标主体属性、背景、布局 | 属性变化渐进、无振荡 |
| **Action Binding** | 指定主体执行了正确动作 | 非目标主体不泄漏/互换动作 | 动作完整连续、非一帧伪成功 |
| **Motion Binding** | 目标主体沿正确方向位移 | 非目标主体/背景不被误带动 | 轨迹平滑、无瞬移抖动 |
| **Spatial Composition** | 目标关系被编辑到位 | 非目标对象/布局不被破坏 | 关系变化过程稳定、无来回翻转 |
| **Background Dynamics** | 目标背景区域正确变化 | 前景主体(运动+外观)+非目标背景 | 变化渐进展开、非闪烁跳变 |
| **View Transformation** | 镜头命令被正确执行 | 主体外观+刚性背景(条件保持) | 镜头平滑、无抖动跳变 |

### 1.5 两种评分聚合模式

根据维度特性，采用两种聚合方式：

**模式 A：线性加权（仅 Attribute Binding 使用）**

```
S_sample = α·E + β·P + γ·C
默认：α=0.50, β=0.20, γ=0.30
```

**模式 B：执行门控式（其余 5 个维度使用）**

```
S_sample = E × (λ·P + (1-λ)·C)
```

各维度默认 λ 值：
- Action Binding: λ=0.60
- Motion Binding: λ=0.60
- Spatial Composition: λ=0.50
- Background Dynamics: λ=0.55
- View Transformation: λ=0.55

**为什么多数维度用执行门控**：如果目标变化根本没做出来（E≈0），那非目标保持再好、过程再平滑，也不该拿到高分。属性维度例外是因为其时序连贯性可以独立于执行正确性评价。

## 二、统一前置流程：实体锚定与跟踪

### 2.1 为什么 Grounding 是前置条件

所有涉及主体的维度，在打分前必须先做 target grounding。这是 I2V 区别于 T2V 评测的核心差异——评测对象必须是**输入图像中的那个实例**，而不是视频里任意一个相似主体。

### 2.2 统一四步流程

**Step 0: Prompt Parsing**

从文本中抽出：
- target entities（目标实体）
- target attributes / actions / motions / relations
- non-target invariants（不应变化的保持约束）

**Step 1: Initial Grounding on I**

用 grounding detector + segmentor 在输入图像上找出目标实例与非目标实例：

```
M = {m₁, …, mₙ}   每个 mᵢ = 实例 mask / bbox / track seed
```

**Step 2: Video Tracking**

在 V 上追踪这些实例，得到轨迹与实例级特征：

```
τᵢ = {mᵢ¹, …, mᵢᵀ}   每帧的 mask/bbox/crop
```

**Step 3: Constraint-specific Evaluation**

根据维度不同，对目标实例或背景区域计算 E、P、C 三项。

### 2.3 统一失败诊断

对每个样本额外标记三类错误（不参与主分数，但在结果分析里单独统计）：

| 错误类型 | 定义 | 示例 |
|----------|------|------|
| **Localization Failure** | 目标实例找错 | 把右边狗当成左边狗 |
| **Binding Failure** | 目标找对了，但变化没执行对 | 找对了狗但颜色没变 |
| **Boundary Failure** | 目标变化基本对，但污染了非目标区域 | 只该变左狗，右狗也变了 |

### 2.4 得分标尺

所有子评分统一映射到 **[0, 1]** 区间：

| 分数段 | 含义 |
|--------|------|
| 0.0 - 0.2 | 完全失败 / 严重偏离 |
| 0.2 - 0.4 | 部分完成但有明显错误 |
| 0.4 - 0.6 | 基本完成但有可察觉缺陷 |
| 0.6 - 0.8 | 良好完成，轻微瑕疵 |
| 0.8 - 1.0 | 优秀 / 近乎完美 |

## 三、评估器总览与技术路线

### 3.1 业界主流评估器分类

| 类型 | 代表工具 | 擅长场景 | 缺点 |
|------|----------|----------|------|
| **MLLM-based** | GPT-4o / Gemini 2.5 / LLaVA-Video / Qwen2.5-VL | 语义理解、动作识别、属性/状态判定、推理 | 成本高、非确定性、几何弱 |
| **Detection-based** | GroundingDINO + SAM2 + Depth Anything V2 | 物体检测、空间关系几何验证、深度估计 | 无法理解语义变化 |
| **Tracking-based** | CoTracker3 / DOT / RAFT 光流 | 运动方向、轨迹分析、位移量化、相机估计 | 短距运动可能信号弱 |
| **Feature-similarity** | DINOv2 / CLIP embedding cosine | 主体一致性、背景一致性、外观保真 | 粒度不够细 |
| **VQA/Classifier** | BLIP-2 VQA / 属性分类器 / Pose Estimator | 属性问答、状态判定、姿态识别 | 受问题设计影响大 |

### 3.2 各基准的评估器选型参考

| 基准 | 维度 | 评估器 |
|------|------|--------|
| T2V-CompBench | Consistent Attr | MLLM (LLaVA-Video image grid) |
| T2V-CompBench | Dynamic Attr | MLLM (LLaVA-Video 6-frame) |
| T2V-CompBench | Spatial Rel | Detection (GroundingDINO + SAM + depth) |
| T2V-CompBench | Motion Binding | Tracking (GroundingSAM + DOT optical flow) |
| T2V-CompBench | Action Binding | MLLM (LLaVA-Video) |
| T2V-CompBench | Object Interaction | MLLM (LLaVA-Video) |
| T2V-CompBench | Numeracy | Detection (GroundingDINO counting) |
| T2I-CompBench++ | Attr Binding | BLIP-VQA (disentangled) |
| T2I-CompBench++ | Spatial Rel | UniDet (2D/3D) |
| T2I-CompBench++ | Non-spatial Rel | CLIPScore |
| T2I-CompBench++ | Complex | 3-in-1 (CLIPScore + BLIP-VQA + UniDet 平均) |
| VBench-I2V | I2V Subject | DINOv2 feature cosine (首帧 vs 各帧 subject crop) |
| VBench-I2V | I2V Background | DINOv2/CLIP feature cosine (首帧 vs 各帧 background) |
| VBench-I2V | Camera Motion | 光流分析 (RAFT) 判定运镜类型 |
| UI2V-Bench | Spatial Understanding | MLLM instance-level evaluation pipeline |
| UI2V-Bench | Attribute Binding | MLLM instance-level evaluation pipeline |
| UI2V-Bench | Reasoning | MLLM feedback-based chain-of-reasoning |

### 3.3 我们的评估器选型原则

**核心原则：维度特性决定主评估器，而非一刀切**

| 维度 | 主评估器 family | 原因 |
|------|----------------|------|
| Attribute Binding | Grounding + Tracking + **frame-wise attribute/state judge** (VLM/classifier) | 属性是视觉语义状态，需逐帧在 subject crop 上判定 |
| Action Binding | Grounding + Tracking + **action-type-aware recognizer / VLM** | 动作需要 subject tube 级时序证据 |
| Motion Binding | Grounding + **Tracking + relative-motion analysis** (CoTracker+RAFT) | 方向是几何问题，不是语义问答 |
| Spatial Composition | **Detection + Depth + geometric rules** | 空间关系是几何判定 |
| Background Dynamics | **多支路**: region motion + photometric + VLM reasoning | 需区分相机运动/区域动态/全图滤镜 |
| View Transformation | **Global motion estimation + geometric analysis** | 镜头是全局几何和轨迹问题 |

**通用规则**：
1. **几何/轨迹类问题不用 MLLM 主判**：Motion、Spatial、Camera 三个维度以 Detection/Tracking 为主
2. **语义/状态类问题以 VLM 为主**：Attribute、Action 以 MLLM/VLM 判定为主
3. **P 轴（保持）偏 Feature Similarity**：保持性可用特征相似度客观量化
4. **MLLM 在几何维度仅作辅助核验**：难例语义裁决、anti-shortcut 检测
5. **所有维度共享统一前置 Grounding + Tracking**：实例锚定是 I2V 评测的刚需

## 四、六维度评测流程详细设计

### 4.1 Attribute / State Transition Binding

#### 定义

测的是：输入图像中已存在的目标主体实例，是否被正确施加了 prompt 指定的属性或状态变化，并且这种变化是否准确绑定到了正确主体上。

**重要原则**：静态属性在 I2V 中不再作为主任务，而作为 preservation constraint 纳入保持分 (P)。

#### 评估器组合

Grounding + Tracking + frame-wise attribute/state judge

- **GroundingDINO / Grounded SAM 2**：输入图像中锚定目标实例
- **SAM 2 video tracking**：视频中持续跟踪，提供每帧 subject crop
- **属性/状态判定器**：
  - 低层属性（颜色、亮度）：mask 区域统计 / 规则判定
  - 高层语义状态（empty/full, open/closed, wet/dry）：image-LLM / VLM judge
  - 采用离散选项式判题，例如 "Is the cup empty or full?"

#### 评分公式

**执行分 E_attr**：

```
E_attr = (1/N) × Σ (1/|𝓜ᵢ¹|) × Σ 𝟙[âᵢ,ₐᵀ = a¹]
```

只衡量该变的属性有没有在正确主体上变对。

**保持分 P_attr**：

```
P_attr = 1 - (1/N) × Σ (1/|Uᵢ|) × Σ Δᵢ(u)
```

其中 Uᵢ = 主体 eᵢ 上未被要求变化的属性集合，Δᵢ(u) = 属性 u 的变化幅度/污染程度。

**连贯分 C_attr**：

```
C_attr = (1/N) × Σ TemporalCoherence(âᵢ,₁:ᵀ; aᵢ⁰, aᵢ¹)
```

检查：首帧接近初态、末帧接近终态、中间帧合理过渡、无来回跳变。

**最终单样本分数（线性加权）**：

```
S_attr = 0.50·E_attr + 0.20·P_attr + 0.30·C_attr
```

#### 难度分桶：S×A 矩阵

S = 主体数，A = 每主体变化属性数

| 桶 | 定义 | v1 保留 |
|----|------|------|
| S1A1 | 单主体，单属性变化 | ✓ |
| S1A2 | 单主体，双属性变化 | ✓ |
| S2A1 | 双主体，单属性变化 | ✓ |
| S2A2 | 双主体，双属性变化 | ✓ |
| S3A1 | 三主体，单属性变化 | 可选 |

#### 失败模式

1. Wrong subject binding：变化发生在错误主体上
2. Transition not reached：终态未达到
3. Attribute leakage：目标变化扩散到其他主体
4. Unspecified attribute corruption：没要求变的属性被改坏
5. Oscillating transition：变化过程来回反复
6. False success by ambiguity：模型生成模糊中间态规避判断

### 4.2 Subject Action Binding

#### 定义

测的是：输入图像中已存在的目标主体实例，是否在视频中执行了 prompt 指定的语义动作，并且动作是否绑定到了正确主体上。非目标主体不应错误执行该目标动作或产生具有混淆性的竞争动作。

**实验目的**：不是测“模型会不会生成动作”，而是测“指定动作是否发生在指定主体上”。

#### 两类子集

- **校准子集 (Calibration Split)**：单主体单动作，测基础动作生成能力，单独报告，不作为 binding 主榜核心
- **主评测子集 (True Binding Split)**：多主体、干扰主体、动作混淆场景，真正测 binding

#### 评估器组合

Grounding + Tracking + action-type-aware recognizer

- **Grounded SAM 2**：实例锚定 + subject tube 生成
- **动作证据提取**（按动作类型分）：
  - 姿态型动作（坐下、站起、跳起）：pose estimator / skeleton trajectory
  - 局部肢体动作（挥手、拍手）：keypoint 时序变化 / video VLM on subject crop
  - 头部动作（点头、转头）：head region evidence
- **非目标动作泄漏检测器**：只惩罚“非目标主体执行了目标动作”，不要求绝对静止

#### 评分公式（执行门控式）

**执行分 E_act**：

```
E_act = (1/|𝒯_act|) × Σ ActionMatch(eᵢ, ψᵢ; V)
```

ActionMatch 基于 subject tube 的时序动作证据，而非单帧分类。

**保持分 P_act（动作泄漏控制）**：

```
P_act = 1 - (1/(|𝒯_act|·(|ℰ|-1))) × ΣΣ ActionLeak(eⱼ, ψᵢ; V)
```

惩罚：非目标主体错误执行了目标动作 / 动作互换 / 动作扩散。

**连贯分 C_act**：

```
C_act = (1/|𝒯_act|) × Σ TemporalValidity(eᵢ, ψᵢ; V)
```

检查：动作是否在连续窗口中成立、是否完整、是否不是一闪而过的伪动作。

**最终单样本分数**：

```
S_act = E_act × (0.6·P_act + 0.4·C_act)
```

#### 难度分桶：S×D 矩阵

S = 主体绑定负载，D = 动作区分难度

| 桶 | 定义 | v1 保留 |
|----|------|------|
| S1D1 | 单主体单动作（校准） | ✓ |
| S2D1 | 双主体，一动一静 | ✓ |
| S2D2 | 双主体，双不同动作 | ✓ |
| S2D3 | 双主体高混淆（同类主体、对称布局） | ✓ |
| S3D2 | 三主体中等负载 | 可选 |

#### 辅助分析指标

- **Execution-Only Accuracy**：单主体校准子集上报告，反映基础动作生成能力
- **Action Swap Rate (ASR)**：多主体子集上报告，越高说明模型越容易动作串台

### 4.3 Subject Motion Binding

#### 定义

测的是：输入图像中已存在的指定主体实例，是否被正确赋予了整体轨迹级的绝对方向运动，并且这种运动应在剔除相机诱导表观位移后成立。

**只测单主体、六个 canonical directions**：left / right / up / down / toward camera / away from camera。不测局部部件动作，不测多主体相对关系变化。

#### 评估器组合（主用 Tracking，不用 MLLM 主判）

**主线：Grounded SAM 2 + CoTracker3 + RAFT**

- Grounded SAM 2：首帧实例锚定 + 视频 mask 传播
- CoTracker3：目标实例区域内稠密点追踪，求平均前景运动向量
- RAFT：全局光流，用于背景主运动估计和相机补偿

**相机运动校正（关键步骤）**：

```
Δp_rel = Δp_fg - Δp_bg
```

前景相对背景的相对位移才是真实运动方向，避免相机运动幻觉。

#### 评分公式（执行门控式）

**执行分 E_mot**：

```
E_mot = DirTrajMatch(e*, μ; I, V)
     = 𝟙[dir correct] × g(|Δp_rel|) × h(depth/scale cue)
```

其中：
- 𝟙[dir correct]：方向符号是否正确
- g(·)：位移量是否足够明显（避免“几乎没动但方向符号碰巧正确”）
- h(·)：仅 toward/away 用，结合 bbox 面积变化或深度估计

**保持分 P_mot（误带动控制）**：

```
P_mot = 1 - (1/|𝒦_mot|) × Σ MotionLeak(q; I, V)
```

惩罚：非目标主体也跟着动了、背景整体被拖拽、相机运动未被正确抵消。

**连贯分 C_mot**：

```
C_mot = PathCoherence × NoTeleport
```

检查：轨迹是否平滑连续、是否存在异常大跳或强烈抖动。

**最终单样本分数**：

```
S_mot = E_mot × (0.6·P_mot + 0.4·C_mot)
```

#### 辅助分析指标

- **Acc_δ**：方向正确且位移≥δ 的样本比例（归一化 100×100 坐标系，默认 δ=5，约 5% 画面位移）
- **Motion Level**：平均相对位移量

### 4.4 Subject Relation / Spatial Composition

#### 定义

测的是：输入图像中已存在的多个主体实例，是否被正确编辑为 prompt 指定的目标相对空间状态。核心不是“原关系是否继续保持”，而是“关系状态是否被正确重排到目标关系”。

v1 核心任务：Pairwise Relation Transition（六类关系：left of / right of / above / below / in front of / behind）

#### 评估器组合（主用 Detection，不用 MLLM 主判）

**主线：GroundingDINO + SAM 2 + Depth Anything V2**

- GroundingDINO：开集文本引导实例检测
- SAM 2：首帧 mask 初始化 + 视频级跟踪
- Depth Anything V2：in front of / behind 的 3D 关系判定

关系判定规则（同 T2V-CompBench）：
- 2D：基于 bbox center，例如 "A left of B" 要求 x(A) < x(B) 且 |x差| > |y差|
- 3D：加 depth estimation 辅助

#### 评分公式（执行门控式）

**执行分 E_rel**：

```
E_rel = (1/|𝒯_rel|) × Σ RelTransitionMatch(ρₘ; I, V)

RelTransitionMatch = 0.2·S_start + 0.5·S_end + 0.3·S_transition
```

最重要的是终态关系是否到位 (S_end)，其次是变化过程 (S_transition)。

**保持分 P_rel**：

```
P_rel = 1 - (1/|𝒦_rel|) × Σ LayoutCorruption(q; I, V)
```

惩罚：无关主体被拖动、未指定 pair 的关系被打乱、背景结构被不必要扭曲。

**连贯分 C_rel**：

```
C_rel = (1/|𝒯_rel|) × Σ TransitionConsistency⁽ᵐ⁾
```

检查：关系是否来回翻转、首尾对但中间乱跳、tracking collapse 造成伪变化。

**最终单样本分数**：

```
S_rel = E_rel × (0.5·P_rel + 0.5·C_rel)
```

#### 辅助分析指标

- **TSR (Target-State Reachability)**：目标关系最终是否被成功达到的比例

### 4.5 Scene / Background Dynamics

#### 定义

测的是：输入图像中已存在的背景区域，是否被正确施加了 prompt 指定的环境动态或场景状态变化。同时前景主体、未指定变化的背景区域应保持稳定。

**实验目的**：不是测“视频是不是更生动了”，而是测 **image-anchored, region-selective environmental control**。

两类任务：
- **动态背景纹理 (Dynamic Texture)**：云流动、海浪增强、烟雾扩散、树叶摇晃
- **场景状态转变 (Scene-State Transition)**：天空变暗、远处起雾、场景变暗淡

#### 评估器组合（四支路管线）

1. **区域锚定 + 视频传播**：Grounded SAM 2 建立目标/非目标区域 mask
2. **Motion/Texture 分析（动态纹理类）**：目标区域内的 camera-compensated residual motion
3. **Photometric/State 分析（状态变化类）**：目标区域亮度/对比度/可见度时序趋势
4. **VLM Reasoning 核验（因果链题）**：question chain 语义確认，防全图滤镜伪成功

**相机运动剔离（关键）**：

```
Δp_res = Δp_obs - Δp_cam
```

必须避免把镜头平移造成的背景表观运动误判为背景动态成功。

#### 评分公式（执行门控式）

**执行分 E_bg**：

```
BGChangeMatch(βₘ) = S_vis × ((1-ρ)·S_loc + ρ·S_sem)

E_bg = (1/|𝒯_bg|) × Σ BGChangeMatch(βₘ)
```

其中：
- S_vis：变化是否可见且达到强度要求
- S_loc：变化是否主要落在目标区域（防全图滤镜作弊）
- S_sem：语义上是否真的构成 prompt 指定的环境变化
- ρ：reasoning 权重，纯动态纹理题较低，因果链题较高

**保持分 P_bg（三支路拆开）**：

```
P_bg = w₁·P_fg-mot + w₂·P_fg-app + w₃·P_bg-static
```

- P_fg-mot：前景主体有没有被无端带动/漂移
- P_fg-app：前景主体外观有没有被改色/改材质/改身份
- P_bg-static：未指定变化的背景区域有没有被错误污染（但允许物理合理的次级光照影响）

**连贯分 C_bg**：

```
C_bg = (1/|𝒯_bg|) × Σ TemporalSceneCoherence⁽ᵐ⁾
```

检查：变化是否渐进展开、是否闪烁、是否先变后回退。

**最终单样本分数**：

```
S_bg = E_bg × (0.55·P_bg + 0.45·C_bg)
```

#### 难度分桶：三轴设计

| 轴 | 低 | 中 | 高 |
|-----|-----|-----|-----|
| 场景复杂度 | 背景开阔、前景少 | 前景干扰存在 | 层次多、遮挡多 |
| 变化复杂度 | 单一动态纹理/单调状态 | 单区域强变化 | 多区域复合变化 |
| Reasoning 负载 | 直接可观察 | 需区分局部/整图 | 有因果链 |

### 4.6 Camera / View Transformation

#### 定义

测的是：模型是否把 prompt 指定的镜头运动或视角变化，正确施加到了输入图像上。同时，未被要求变化的主体、主体属性和刚性背景内容，仅应发生由镜头变化必然带来的投影变化，而不应被额外改写。

**关键设计原则**：
- "static camera" 是执行目标，不是保持项
- 必须显式区分刚性背景和内生动态背景
- v1 主 benchmark 优先选刚性背景样本

#### 任务形式

1. **Single Camera Command**：zoom in/out, pan left/right, tilt up/down, static
2. **Static Camera under Scene Motion**（负控制）：主体在动但镜头必须保持静止
3. **Camera + Framing Constraint**：zoom in + keep subject centered

#### 评估器组合（主用几何分析，不用 MLLM 主判）

**主线：Grounded SAM 2 + CoTracker/RAFT + crop-based visual judge**

- Grounded SAM 2：实例/背景区域划分 + 视频传播
- RAFT / OpenCV global motion fitting：基于背景稳定区域估计全局相机运动
- Crop-based visual judge：辅助判定属性是否漂移、背景是否被内容性改写

#### 评分公式（执行门控式）

**执行分 E_cam**：

```
E_cam = (1/|𝒯_cam|) × Σ CamMatch(K₁:ᵀ, κₘ)
```

CamMatch 检查：运镜类型正确、方向正确、趋势持续存在、构图约束满足。

**保持分 P_cam（三支路）**：

```
P_cam = 1 - [λ_o·ΣObjResidualDrift + λ_a·ΣAttrCorruption + λ_b·ΣRigidBgWarpErr]
默认：λ_o=0.4, λ_a=0.3, λ_b=0.3
```

- ObjResidualDrift：相机补偿后未要求运动的对象是否仍有残差漂移
- AttrCorruption：未要求变化的主体属性是否漂移
- RigidBgWarpErr：刚性背景在 warp 对齐后是否仍有内容性残差

**连贯分 C_cam**：

```
C_cam = (1/3) × (NoJitter + NoAbruptJump + TrendConsistency)
```

**最终单样本分数**：

```
S_cam = E_cam × (0.55·P_cam + 0.45·C_cam)
```

#### 难度分桶：场景复杂度 × 运镜复杂度

| 桶 | 定义 | v1 保留 |
|----|------|------|
| S1C1 | 单主体+单镜头命令 | ✓ |
| S2C1 | 双主体/干扰+单命令 | ✓ |
| S1C2 | 单主体+镜头+构图约束 | ✓ |
| S2C2 | 双主体+镜头+构图约束 | ✓ |
| S3C3 | 多主体+复合镜头 | 可选 |

背景策略标签：R=刚性背景（主 benchmark），D=动态背景（stress subset）

#### 辅助分析指标

- **CSR (Camera Command Success Rate)**：镜头命令执行成功率
- **False Camera Motion Rate**：static-camera 样本中被判定存在显著相机运动的比例
- **Static Content Integrity**：camera-only 样本中未指定变化内容的保持度

### 5.1 统一实例 Grounding 模块

```python
def unified_grounding(image, text_entities):
    """
    Step 0-1: 在输入图像上锚定所有目标与非目标实例
    所有 6 个维度共享这一前置步骤
    """
    # GroundingDINO: 文本引导的开集检测
    bboxes = grounding_dino.detect(image, text_entities)
    # SAM 2: 获取精确 mask
    masks = sam2.segment(image, bboxes)
    return {entity: mask for entity, mask in zip(text_entities, masks)}
```

### 5.2 统一视频 Tracking 模块

```python
def unified_tracking(video_frames, initial_masks):
    """
    Step 2: 在视频上追踪所有实例
    SAM 2 Video + CoTracker3 组合
    """
    # SAM 2 Video: mask propagation 跟踪 mask/bbox
    tracked_masks = sam2_video.track(video_frames, initial_masks)
    # CoTracker3: 密集点追踪（用于运动/轨迹分析）
```

### 5.3 主体一致性评估器 (P 轴通用)

```python
def eval_subject_preservation(first_frame, video_frames, subject_mask):
    """
    基于 DINOv2 的主体外观一致性 — 用于 P 轴
    参考: VBench-I2V i2v_subject
    """
    ref_feature = dinov2.extract(crop(first_frame, subject_mask))
    scores = []
    for frame, mask in zip(video_frames, tracked_masks):
        frame_feature = dinov2.extract(crop(frame, mask))
        scores.append(cosine_similarity(ref_feature, frame_feature))
    return mean(scores)
```

### 5.4 背景一致性评估器 (P 轴通用)

```python
def eval_background_preservation(first_frame, video_frames, all_masks):
    """
    基于 DINOv2 的背景区域一致性 — 用于 P 轴
    mask 掉所有主体后对比背景
    """
    ref_bg = mask_out(first_frame, all_masks[0])
    ref_feature = dinov2.extract(ref_bg)
    scores = []
    for frame, mask in zip(video_frames, all_masks):
        frame_bg = mask_out(frame, mask)
        scores.append(cosine_similarity(ref_feature, dinov2.extract(frame_bg)))
    return mean(scores)
```

### 5.5 相机运动估计模块

```python
def estimate_camera_motion(video_frames, background_masks):
    """
    基于 RAFT + 背景区域主运动估计
    用于 Motion/Camera/Background 三个维度的相机补偿
    """
    flows = [raft.compute(video_frames[t], video_frames[t+1])
             for t in range(len(video_frames)-1)]
    # 只取背景区域的光流作为相机运动估计
    cam_motion = [mean_flow_in_mask(f, bg_mask) for f, bg_mask in zip(flows, background_masks)]
    return cam_motion

def compute_relative_motion(fg_motion, cam_motion):
    """ Δp_rel = Δp_fg - Δp_bg """
    return fg_motion - cam_motion
```

### 5.6 时序连贯性评估器 (C 轴通用)

```python
def eval_temporal_coherence(signal_sequence, mode="monotonic"):
    """
    检测时序信号是否平滑/单调/无抖动
    用于 C 轴的通用模块
    """
    # 检测突变
    diffs = [abs(signal_sequence[t+1] - signal_sequence[t])
             for t in range(len(signal_sequence)-1)]
    no_jitter = 1.0 - max(diffs) / tolerance  # 无剧烈跳变
    # 检测单调性（如适用）
    if mode == "monotonic":
        monotonic_score = count_monotonic_pairs(signal_sequence) / total_pairs
        return (no_jitter + monotonic_score) / 2
    return no_jitter
```

## 八、聚合与报告

### 8.1 四层聚合结构

```
Level 1: Per-sample score
    S_i = 按维度聚合公式（线性加权 or 执行门控）
    同时记录 E_i, P_i, C_i 三分和 failure_type 诊断

Level 2: Per-bucket score（难度桶级别）
    S_bucket = mean(S_i for i in bucket)
    E_bucket = mean(E_i for i in bucket)
    P_bucket = mean(P_i for i in bucket)
    C_bucket = mean(C_i for i in bucket)

Level 3: Per-dimension score（维度级别）
    S_dim = mean(S_bucket for all buckets)  # 或难度加权
    E_dim = mean(E_bucket)
    P_dim = mean(P_bucket)
    C_dim = mean(C_bucket)

Level 4: Overall score（模型总分）
    S_overall = α·E_overall + β·P_overall + γ·C_overall
    推荐默认：α=0.45, β=0.35, γ=0.20
    其中 E/P/C_overall = mean(across 6 dimensions)
```

| Q8 | Camera 维度中动态背景样本处理 | R=刚性背景为主评测，D=动态背景为 stress subset | 分开报告 |

## 十二、VLM Prompt 模板库（草稿）

> 以下模板仅用于需要 VLM 判定的语义性任务（Attribute/Action 的 E 轴、Background 的 S_sem）。
> 几何/轨迹类维度（Motion/Spatial/Camera）不使用 VLM 主判，只在 anti-shortcut 核验时调用。

### 12.1 Attribute State Judge Prompt

```
You are evaluating whether a specific attribute/state change has been applied to a target subject in an I2V video.

【Subject & Change】
- Target subject: {target_subject}
- Expected change: {initial_state} → {target_state}
- Frame position: This is frame {t} of {T}

【Task】
Look at the cropped subject region and answer:
1. Current state of {attribute_name}: [choose from: {initial_state} / {target_state} / intermediate / ambiguous]
2. Confidence: [high / medium / low]

Return ONLY a JSON: {"state": "...", "confidence": "..."}
```

### 12.2 Action Evidence Prompt (tube-level)

```
You are evaluating whether a specific subject performs a specified action in a video.

【Subject & Action】
- Target subject: {target_subject}
- Expected action: {target_action}
- These {N} frames show ONLY the cropped region of the target subject across time.

【Task】
1. Does the subject appear to be performing "{target_action}"? [yes / partially / no]
2. Is the action complete and temporally coherent? [yes / fragmented / static pose only]
3. Could this be a single-frame illusion rather than actual motion? [yes / no]

Return JSON: {"action_match": "...", "temporal_validity": "...", "illusion_risk": "..."}
```

### 12.3 Background Anti-Shortcut Prompt

```
You are checking whether a background change in an I2V video is a genuine regional environmental change, or just a global filter/color shift applied to the entire frame.

【Expected Change】: {expected_bg_change}
【Target Region】: {target_region_description}

【Evidence Frames】: {N} frames from the video

【Task】
1. Is the change localized to the target region, or does it affect the entire frame equally? [localized / global / mixed]
2. Are foreground subjects affected by the change? [unaffected / slightly affected / strongly affected]
3. Does the change semantically match "{expected_bg_change}"? [yes / partially / no]

Return JSON: {"locality": "...", "fg_impact": "...", "semantic_match": "..."}
```

### 12.4 P 轴通用 Preservation Spot-Check Prompt

```
You are checking whether non-target elements in an I2V video have been preserved.

【Context】
- The video was supposed to ONLY change: {change_description}
- These elements should remain UNCHANGED: {preserve_list}

【Two frames】: Frame 1 (first frame) and Frame 2 (last frame)

【Task】
For each element in the preserve list, report:
- Status: [preserved / slightly changed / significantly changed / missing]
- If changed, describe what changed.

Return JSON array: [{"element": "...", "status": "...", "note": "..."}]
```
