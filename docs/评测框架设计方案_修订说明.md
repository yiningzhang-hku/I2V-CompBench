# 评测框架设计方案 修订说明

> 本文档记录旧版「评测框架设计方案.md」相对于「关于评测的思考.docx」中完整框架的不足之处，以及本次修订的对齐内容。

---

## 一、总体架构层面的不足

### 1.1 三轴命名与语义错误

| 项 | 旧版（错误） | 正确定义 |
|----|-------------|---------|
| E | Evaluation（综合得分） | **Exec（执行分）**：目标变化有没有执行对 |
| P | Performance（变化轴得分） | **Preserve（保持分）**：未指定变化的部分有没有保持 |
| C | Consistency（保持轴得分） | **Coherence（连贯分）**：变化过程是否时序、几何、物理合理 |

旧版实质上只有两个正交轴（变化/保持），缺少第三轴 Coherence。Coherence 不可归入 E 或 P——"首尾碰巧正确但中间乱闪"的视频 E 可能高但 C 该低，"过程平滑但方向错"的视频 C 高但 E 低。

### 1.2 评分聚合公式错误

| 项 | 旧版 | 正确 |
|----|------|------|
| 统一公式 | `E = 0.6P + 0.4C`（所有维度相同） | 两种模式，按维度区分 |
| Attribute | 同上 | **线性加权** `S = 0.50E + 0.20P + 0.30C` |
| Action | 同上 | **执行门控** `S = E × (0.6P + 0.4C)` |
| Motion | 同上 | **执行门控** `S = E × (0.6P + 0.4C)` |
| Spatial | 同上 | **执行门控** `S = E × (0.5P + 0.5C)` |
| Background | 同上 | **执行门控** `S = E × (0.55P + 0.45C)` |
| Camera | 同上 | **执行门控** `S = E × (0.55P + 0.45C)` |
| Overall | `E_overall = mean(E_dim)` | `S = 0.45E + 0.35P + 0.20C` |

**执行门控的核心逻辑**：如果目标变化根本没做出来（E≈0），那非目标保持再好、过程再平滑也不应拿到高分。Attribute 例外是因为其时序连贯性可以独立于执行正确性评价。

### 1.3 缺少统一前置 Grounding 流程

旧版各维度各自提及 GroundingDINO/SAM2，但没有将其抽象为**所有维度共享的前置条件**。

正确架构应为统一四步：
1. **Step 0: Prompt Parsing** — 抽取 target entities, changes, invariants
2. **Step 1: Initial Grounding on I** — 输入图像上实例锚定
3. **Step 2: Video Tracking** — 视频级实例跟踪
4. **Step 3: Constraint-specific Evaluation** — 维度专属评测

这是 I2V 区别于 T2V 评测的核心差异——评测对象必须是**输入图像中的那个实例**，而不是视频里任意一个相似主体。

### 1.4 缺少统一失败诊断

旧版没有失败分类。正确框架应额外标记三类错误（不参与主分数，但在结果分析里单独统计）：

| 错误类型 | 定义 | 示例 |
|----------|------|------|
| Localization Failure | 目标实例找错 | 把右边狗当成左边狗 |
| Binding Failure | 目标找对了，但变化没执行对 | 找对了狗但颜色没变 |
| Boundary Failure | 目标变化基本对，但污染了非目标区域 | 只该变左狗，右狗也变了 |

## 二、评估器选型原则的不足

### 2.1 旧版："MLLM 为主，工具为辅"（错误）

旧版将 MLLM（GPT-4o / Gemini）定位为所有维度的核心判断器，Detection/Tracking 仅作辅助。

### 2.2 正确原则："维度特性决定主评估器"

| 维度 | 正确主评估器 | 旧版错误 |
|------|-------------|---------|
| Attribute | Grounding + Tracking + frame-wise VLM/classifier | 旧版用 MLLM 做整体判断，缺少 subject crop 级逐帧判定 |
| Action | Grounding + Tracking + action-type-aware recognizer | 旧版仅用 MLLM，缺少 pose/keypoint/tube-level 证据 |
| **Motion** | Grounded SAM 2 + CoTracker + RAFT | **旧版用 MLLM 做 secondary，应改为不用 MLLM 主判** |
| **Spatial** | GroundingDINO + SAM 2 + Depth Anything V2 | **旧版用 MLLM 做 secondary，应改为不用 MLLM 主判** |
| **Background** | 多支路：region anchoring + motion/texture + VLM reasoning | 旧版过度依赖 MLLM，缺少区域锚定和 anti-shortcut |
| **Camera** | Grounded SAM 2 + CoTracker/RAFT + crop-based judge | **旧版用 MLLM 做 secondary，应改为不用 MLLM 主判** |

**核心规则**：几何/轨迹类问题（Motion、Spatial、Camera）不用 MLLM 主判，MLLM 在这些维度仅作辅助核验。

## 三、各维度评测流程的不足

### 3.1 Attribute Binding

| 不足项 | 说明 |
|--------|------|
| 缺少"静态属性→P轴"原则 | I2V 中静态属性不再作为主任务，而是纳入 Preservation constraint |
| 评估器粒度不够 | 旧版用整体 MLLM 打分，应改为逐帧在 subject crop 上做离散选项式判定 |
| 缺少 E/P/C 分立公式 | 旧版只有笼统的 P 和 C，缺少形式化的 E_attr / P_attr / C_attr 定义 |
| 缺少难度分桶定义 | 缺少 S×A 矩阵（主体数 × 每主体变化属性数） |
| 缺少失败模式枚举 | 至少 6 种典型失败模式未列出 |

### 3.2 Action Binding

| 不足项 | 说明 |
|--------|------|
| 缺少校准/主评测拆分 | 应将单主体动作作为 Calibration Split，多主体才是 True Binding Split |
| 评估器缺少 action-type 区分 | 不同动作类型（姿态型/局部肢体/头部）需要不同证据源 |
| 非目标约束定义错误 | 旧版要求"非目标保持静止"，正确做法是只惩罚"非目标执行了目标动作"（动作泄漏），不要求绝对静止 |
| 缺少执行门控公式 | 旧版用线性 `E = 0.6P + 0.4C`，应为 `S = E × (0.6P + 0.4C)` |
| 缺少 S×D 难度矩阵 | 主体绑定负载 × 动作区分难度 |
| 缺少 ASR 辅助指标 | Action Swap Rate 未定义 |

### 3.3 Motion Binding

| 不足项 | 说明 |
|--------|------|
| 范围定义过宽 | 旧版包含类型 A/B/C 三种，正确版只测**单主体、六个 canonical directions** |
| **缺少相机运动校正** | 这是最关键的遗漏——`Δp_rel = Δp_fg - Δp_bg`，不做校正会把相机运动误判为主体运动 |
| 评估器错误 | 旧版用 MLLM 作 secondary，正确版完全不用 MLLM 主判 |
| 缺少 DirTrajMatch 公式 | `E = 𝟙[dir correct] × g(|Δp_rel|) × h(depth/scale cue)` |
| 缺少 Acc_δ 辅助指标 | 方向正确且位移≥δ 的样本比例 |

### 3.4 Spatial Composition

| 不足项 | 说明 |
|--------|------|
| **任务定义根本错误** | 旧版测的是"空间关系是否被正确保持"，正确任务是**"关系是否被正确编辑到目标状态"**（Pairwise Relation Transition） |
| 缺少 RelTransitionMatch 公式 | `0.2·S_start + 0.5·S_end + 0.3·S_transition` |
| λ 值错误 | 旧版用 0.6/0.4，正确为 **0.5/0.5**（Spatial 维度中保持和连贯同等重要） |
| 缺少 TSR 辅助指标 | Target-State Reachability |

### 3.5 Background / Scene Dynamics

| 不足项 | 说明 |
|--------|------|
| 缺少两类任务区分 | 应区分动态背景纹理（Dynamic Texture）和场景状态转变（Scene-State Transition） |
| 缺少四支路管线 | 旧版仅用 MLLM + CLIP，正确版需要：区域锚定 + Motion/Texture + Photometric/State + VLM Reasoning |
| **缺少 anti-shortcut 设计** | 最重要的遗漏——必须检测"全图滤镜伪成功"（模型对整帧加色调而非仅在目标区域产生环境变化） |
| **缺少相机运动剥离** | `Δp_res = Δp_obs - Δp_cam`，不做则镜头平移造成的背景表观运动会被误判为成功 |
| P 轴拆分不够细 | 旧版只有 C_fg/C_fg_pos/C_fg_act，正确版拆为 P_fg-mot + P_fg-app + P_bg-static |
| 难度分桶维度不够 | 应为三轴：场景复杂度 × 变化复杂度 × Reasoning 负载 |

### 3.6 Camera / View Transformation

| 不足项 | 说明 |
|--------|------|
| **缺少"static camera 是执行目标"原则** | Static camera 应作为 E 轴评测对象，而非仅作为其他维度的保持约束 |
| 缺少刚性/动态背景区分 | v1 主 benchmark 应优先选刚性背景样本，动态背景作为 stress subset |
| 评估器错误 | 旧版以全局光流分类为主，正确版需要分出前景/背景/刚性区域分别分析 |
| P 轴公式缺失 | 应拆为三路：ObjResidualDrift + AttrCorruption + RigidBgWarpErr |
| 缺少背景策略标签 | R=刚性背景（主 benchmark）、D=动态背景（stress subset） |
| 缺少辅助指标 | CSR（Camera Command Success Rate）、False Camera Motion Rate、Static Content Integrity |

## 四、其他结构性不足

### 4.1 初始状态形式化缺失

旧版缺少对输入图像的形式化表达：

```
C₀ = {E₀, A₀, R₀, S₀, B₀, K₀}
```

其中 E₀=实体集合、A₀=属性集合、R₀=关系集合、S₀=状态集合、B₀=背景/场景状态、K₀=相机/视角状态。文本定义的是目标变化 ΔC，不是重建整个世界。

### 4.2 通用评估器模块不够

旧版只有 Subject/Background/Camera/Temporal 四个通用模块。正确版需要增加：
- **统一 Grounding 模块**（所有维度共享前置）
- **统一 Tracking 模块**（SAM 2 Video + CoTracker3 组合）
- **相机运动估计模块**（Motion/Camera/Background 三维度共用）
- **时序连贯性评估器**（通用 C 轴模块，检测单调性/抖动/跳变）

### 4.3 VLM Prompt 设计思路错误

旧版 prompt 要求 MLLM 对整个视频给 0-100 分，这种方式：
- 粒度太粗，无法区分 E/P/C 三轴
- 几何/轨迹维度不应用 MLLM 主判

正确设计应为：
- 仅在 Attribute/Action 的 E 轴和 Background 的语义核验中使用
- 在 subject crop 级别做离散选项式判定（而非全局打分）
- 返回结构化 JSON 而非自然语言

### 4.4 报告结构不够丰富

旧版缺少：
- 各维度的 aggregation mode 和 λ 值记录
- 各维度的辅助分析指标（ASR, Acc_δ, TSR, CSR 等）
- failure_diagnostics 字段
- 评估器组合的完整记录（grounding_model, tracking_model, vlm_model 分开记录）
