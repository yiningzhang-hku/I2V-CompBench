# TIP-I2V-Prior-Analysis 子工程详细介绍

> I2V-CompBench 的"先验分析"子工程：从真实 I2V 用户提示词数据集中提取结构化先验，驱动下游 Benchmark 题目合成。

---

## 一、项目定位

`tip_i2v_data_analysis` 是 **I2V-CompBench**（图像到视频组合性评测基准）的**上游子系统**。它的职责是从真实世界的 I2V 用户数据中"挖矿"，产出结构化的 **Prior Package（先验包）**，作为下游 LLM 合成评测题目的**唯一数据驱动来源**。

### 在 I2V-CompBench 中的位置

```
TIP-I2V 真实数据集 (HuggingFace tipi2v/TIP-I2V, 181 万条)
        │
        ▼
┌───────────────────────────────────────┐
│  本工程：tip_i2v_data_analysis        │
│  - 5 步流水线（Manifest → VLM → LLM   │
│    → Joint → Prior Package）          │
│  - 只读分析，不做合成                  │
└───────────────────────────────────────┘
        │
        ▼
   prior_package.json  ★ 核心交付物
        │
        ▼
下游工程：LLM 合成全维度 prompt + 文生图合成首帧 → 完整评测基准
```

**关键定位**：本工程**只做分析与先验提取，不参与题目合成**。它是 I2V-CompBench 这台"知识蒸馏机"的前端环节，把真实用户数据中的概念分布、视觉构成规律、句式模板、约束规则蒸馏成下游可直接使用的结构化产物。

### 数据来源

- **数据集**：HuggingFace 上的 [`tipi2v/TIP-I2V`](https://huggingface.co/datasets/tipi2v/TIP-I2V)
- **来源**：Pika Discord 社区的 181 万条真实 I2V 生成记录
- **样本结构**：用户提示词文本 + 首帧缩略图（224px）+ 主体类别字段 + UUID
- **当前使用 split**：Eval（10K 条），典型批次处理 1000 条样本进行验证
- **拉取方式**：HuggingFace `streaming=True` 流式加载，避免下载完整 parquet 文件

## 二、核心评测维度

本工程围绕 **6 个评测维度**进行分析（第 7 维 Interaction Reasoning 列在 [七维度重构方案.md](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/docs/七维度重构方案.md) 中作为后续扩展）：

| # | 维度 | 含义 | 关键输入模式 |
|---|------|------|------|
| 1 | **Subject Attribute Binding** | 主体属性 / 状态变化 | 单图编辑 + 多图属性迁移 |
| 2 | **Subject Action Binding** | 肢体动作的正确绑定 | 单图分配 + 多图场景合成 |
| 3 | **Subject Motion Binding** | 主体轨迹运动（绝对方向 + 相对位移） | 单图 + 多图组合 |
| 4 | **Subject Spatial Composition** | 多参考主体的静态空间布局组合 | 多图为主（80%） |
| 5 | **Scene / Background Dynamics** | 环境状态变化 + 背景迁移 | 单图变化 + 多图替换 |
| 6 | **Camera / View Transformation** | 镜头 / 视角控制 | 单图为主（95%） |

每个维度的可评测性（evaluability）由两个条件共同决定：**文本是否请求** AND **图像是否支持**。详见 [src/step4_joint_analysis.py](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/src/step4_joint_analysis.py) 中的 `DIM_CONSTRAINTS`。

### Step 1 — Manifest 构建

源代码：[src/step1_manifest.py](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/src/step1_manifest.py)

**职责**：
1. 优先尝试 HuggingFace 流式加载（`tipi2v/TIP-I2V`），失败则回退到本地 `parquet/jsonl/csv` 文件
2. 提取并清洗 **Pika 参数**（`-camera`, `-motion`, `-neg`, `-fps`, `-gs`, `-ar`, `-seed`），分离出 `pika_camera` / `pika_motion` 字段
3. 将首帧 PIL 图像转码为 JPEG 落盘到 `manifest_dir/images/`
4. 为每条样本生成 `ManifestItem`，并按 `status` 字段（`ok` / `missing_image` / `empty_prompt` / `bad_format`）分流

**核心函数**：`clean_pika_prompt(raw_prompt)`（位于 [src/utils/io_utils.py](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/src/utils/io_utils.py)）使用正则提取 Pika 参数，支持带引号的负向提示。

**输出**：
- `manifest.jsonl`：全部样本（含坏样本）
- `manifest_clean.jsonl`：仅 `status=ok` 的有效样本
- `bad_samples.jsonl`：失败样本及错误详情
- `images/<sample_id>.jpg`：首帧图像

**性能**：1000 样本约 24 秒。

### Step 2 — VLM 图像结构化分析

源代码：[src/step2_image_analysis.py](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/src/step2_image_analysis.py)

**职责**：调用 VLM（Qwen3-30B-A3B-Instruct-2507）将每张首帧图像解析为严格 JSON：

| 解析维度 | 字段 |
|---------|------|
| 主体（thing 类） | `subjects[]`：name / instance_description / attributes（color/size/material/state/wearing） / pose_action / position_in_frame / is_animate |
| 主体关系 | `subject_relations[]`：subject_a / predicate（11 类空间/接触关系） / subject_b |
| 背景（stuff 类） | `background.elements[]`（rigid / potentially_dynamic）+ lighting / weather / time_of_day / 前后景可分离性 / 刚性背景占比 |
| 运镜基线 | `camera_baseline`：shot_type / framing / camera_angle / depth / has_rigid_reference_structure / scene_depth_complexity |

**关键能力**：
- 异步并发（默认 `batch_size=10`），通过 `asyncio.Semaphore` 限流
- 自动断点续传：通过 `read_processed_ids()` 读取已处理 sample_id，重启自动跳过
- 自动指数退避重试（`tenacity` 库 + 自定义异步重试）
- 图像自动缩放（长边 > 2048px 时缩到 2048px）+ JPEG 转码

**提示词模板**：[prompts/vlm_image_parse.txt](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/prompts/vlm_image_parse.txt)

**性能**：约 25 秒/样本（含 API 延迟与重试），5 并发下 1000 样本约 7 小时。

### Step 3 — LLM 文本意图与语义分析

源代码：[src/step3_text_analysis.py](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/src/step3_text_analysis.py)

**职责**：调用 LLM 对清洁后的 prompt 做**两层分析**：

**第一层 — 意图分类**：
- `primary_intent` ∈ {`subject_focused`, `background_focused`, `camera_focused`, `mixed`, `ambiguous`}
- 三类子意图细分：subject_sub_intent / background_sub_intent / camera_sub_intent

**第二层 — 6 维度语义槽位提取**：

| 槽位 | 字段 |
|------|------|
| `attribute_change_slots` | target_subject / attribute_type（color/texture/state/wearing/size）/ from_value / to_value |
| `action_slots` | target_subject / action_verb / action_detail |
| `motion_slots` | target_subject / direction（8 个 canonical directions）/ motion_phrase |
| `spatial_relation_slots` | subject_a / target_predicate（7 类）/ subject_b |
| `background_change_slots` | target_region / change_type / from_state / to_state |
| `camera_movement_slots` | command（13 类运镜）/ target_subject / speed_modifier / framing_constraint |

外加从原始 prompt 提取的词性标注：`nouns[]` / `verbs[]` / `adjectives[]`。

**`involves_*` 布尔标记**：只要 prompt 涉及该维度就为 true（即使不是主要意图），用于后续多维度共现分析。

**提示词模板**：[prompts/llm_text_parse.txt](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/prompts/llm_text_parse.txt)

**性能**：约 30 秒/样本，5 并发下 1000 样本约 9 小时。

### Step 4 — 联合分析与深度先验提取

源代码：[src/step4_joint_analysis.py](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/src/step4_joint_analysis.py)

这是工程的**核心分析步骤**，包含 4 个子模块：

#### Part A：维度可评测性判定

对每个样本逐维度判定 `evaluable = text_requests AND image_supports`：

| 维度 | text 条件 | image 条件 |
|------|----------|----------|
| attribute_binding | `involves_attribute_change=true` | 至少一个主体有可见属性 |
| motion_binding | `involves_directed_motion=true` | `subject_count >= 1` |
| spatial_relation | `involves_spatial_relation_change=true` | `has_multiple_subjects` AND `subjects_clearly_distinguishable` |
| action_binding | `involves_action=true` | 至少一个 `is_animate=true` 主体 |
| scene_dynamics | `involves_background_change=true` | `foreground_background_separability ∈ {high, medium}` |
| camera_transformation | `involves_camera_movement=true` | `has_rigid_reference_structure=true` |

同时计算文本目标主体与图像检测主体的 `subject_overlap_ratio`，作为对齐质量指标。

#### Part B：概念分布提取

按维度对所有可评测样本的语义槽位进行聚合统计，产出 `ConceptDistribution`（含 count + pct）：

- `attribute_binding`：target_subject、attribute_type、change_pattern（`from -> to`）
- `action_binding`：target_subject、action_verb、action_detail
- `motion_binding`：target_subject、direction
- `spatial_relation`：subject、predicate
- `scene_dynamics`：target_region、change_type、to_state
- `camera_transformation`：camera_command、camera_target_subject

#### Part C：视觉构成先验

从可评测样本的图像分析结果中聚合 6 类视觉特征分布（`VisualCompositionPrior`）：
- `subject_count_distribution`
- `scene_type_distribution`（lighting × weather × time 组合）
- `shot_type_distribution`
- `camera_angle_distribution`
- `background_separability_distribution`
- `typical_subject_categories`

**用途**：指导下游文生图模型生成符合真实分布的首帧。

#### Part D：附加产出

- **Pika 参数分布**：直接以 `pika_camera_distribution` / `pika_motion_distribution` 形式整合，作为相机维度的**真实用户偏好**记录
- **维度共现矩阵**：6×6 矩阵，记录两个维度同时出现的频率，用于多维度组合题设计
- **种子样例**（Seed Examples）：按综合评分（subject_overlap_ratio + 信息丰富度 + parse_success）选 Top-N，作为 LLM few-shot 提示的高质量示例

#### 静态约束规则

[src/step4_joint_analysis.py](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/src/step4_joint_analysis.py) 的 `DIM_CONSTRAINTS` 字典定义了每个维度的**静态约束知识**（最少主体数、合法属性类型、合法运镜命令等），随 Prior Package 一并下发。

### Step 5 — Prior Package 打包 + 报告生成

源代码：[src/step5_pool_and_report.py](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/src/step5_pool_and_report.py)

**职责**：
1. 加载 Step 4 全部中间产出
2. 组装顶层 `PriorPackage` JSON 对象（核心交付物）
3. 渲染可读的 [summary.md](file:///d:/projects/I2V-CompBench/tip_i2v_data_analysis/docs/summary.md) 报告
4. 输出辅助 CSV 统计表

**核心交付物 — `prior_package.json` 顶层结构**：

```jsonc
{
  "dataset_name": "TIP-I2V",
  "split": "Eval",
  "total_samples": 1000,
  "clean_samples": 989,
  "analyzed_samples": 950,

  "global_distributions": [...],         // 全局意图、名词、动词、形容词分布
  "global_visual_prior": {...},          // 全局视觉构成分布
  "pika_camera_distribution": [...],     // 用户运镜命令偏好
  "pika_motion_distribution": [...],     // 用户 motion 级别偏好
  "dimension_priors": [...],             // 6 个维度独立先验（核心）
  "dimension_cooccurrence": {...}        // 6×6 共现矩阵
}
```

每个 `DimensionPrior` 又包含：`concept_distributions` + `visual_prior` + `structural_templates` + `seed_examples` + `constraints`，是下游 LLM 合成的完整指令源。

`SiliconFlowClient` 类封装了 SiliconFlow 的 OpenAI 兼容接口，提供：
- 同步：`call_vlm()` / `call_llm()`

- `tenacity` 同步重试 + 自定义异步指数退避重试
- 速率限制（`rate_limit_delay`）

## 九、设计亮点

1. **流式数据获取**：通过 HuggingFace `streaming=True` 避免下载完整 1.7M 数据
2. **断点续传**：每步通过 `read_processed_ids()` 自动跳过已处理样本，长时任务可中断重启
3. **结构化契约**：所有跨步骤数据通过 Pydantic v2 模型定义，强类型约束防止 schema 漂移
4. **异步并发 + 重试**：VLM/LLM 调用使用 `asyncio` + `Semaphore` 限流，配合指数退避重试，对抗网络抖动与 API 限流
5. **配置驱动**：所有路径、阈值、API 参数集中在单一 YAML，易于复现与多次迭代
6. **Pika 参数白盒提取**：用户在 prompt 中夹带的 `-camera`/`-motion` 等运镜偏好被精准识别并独立统计，作为 camera 维度的真实先验
7. **维度可评测性双约束**：`text_requests AND image_supports` 双门槛保证统计意义上的样本质量
8. **种子样例机制**：每维度筛选 Top-N 高质量示例，直接服务下游 LLM few-shot
