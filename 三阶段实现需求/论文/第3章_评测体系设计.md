# 第3章 I2V-CompBench任务与标注体系设计

## 3.1 研究目标与范围

本文面向文本指令驱动的图像到视频生成（Image-to-Video, I2V）任务，研究如何构建具有诊断能力的组合性评测数据集。与以画面质量、运动平滑度或整体图文相似度为核心的评测不同，本文关注的问题是：在输入图像已经给定主体、外观、布局和背景的条件下，模型能否只改变文本指令指定的内容，并保持其他视觉条件稳定。

为保证论文贡献与工程完成度一致，本文正式数据集限定为五个已完成维度：属性绑定、动作绑定、运动绑定、背景动态和视角变换。空间组合与交互推理需要稳定的多主体定位、多图身份绑定和因果时序判定，当前产物尚不足以支持可靠评测，因此不纳入本文正式版本，也不计入样本总量。最终数据集计划从质量合格候选池中为每个维度分层筛选300条，共1500条单图I2V评测样本。

本文负责评测任务、样本约束和结构化标注的设计。Execution、Preservation和Coherence（E/P/C）的具体自动评分器由后续评测模块实现，本章仅定义三类信息在数据样本中的语义及其接口要求。

## 3.2 I2V组合性任务形式化

### 3.2.1 条件生成任务

设输入首帧为 $I_0$，文本指令为 $P$，I2V模型为 $G_\theta$，生成视频表示为：

$$
V=G_\theta(I_0,P)=\{f_1,f_2,\ldots,f_T\},
$$

其中 $f_t$ 为第 $t$ 帧。输入图像定义初始视觉状态：

$$
C_0=(\mathcal{E}_0,\mathcal{A}_0,\mathcal{R}_0,\mathcal{S}_0,\mathcal{B}_0,\mathcal{K}_0),
$$

其中，$\mathcal{E}_0$ 表示实体集合，$\mathcal{A}_0$ 表示实体属性，$\mathcal{R}_0$ 表示实体间关系，$\mathcal{S}_0$ 表示动作与运动状态，$\mathcal{B}_0$ 表示背景环境，$\mathcal{K}_0$ 表示相机状态。文本指令并非要求重新生成全部状态，而是规定其中一部分状态的目标变化。

### 3.2.2 Preserve-Transform双轴约束

本文将每条I2V指令表示为变化约束集合 $\mathcal{T}$ 与保持约束集合 $\mathcal{P}$：

$$
\mathcal{T}=\{(z,z')\mid z\in C_0,\;z'\text{为文本指定的目标状态}\},
$$

$$
\mathcal{P}=\{z\in C_0\mid z\notin \operatorname{dom}(\mathcal{T})\}.
$$

$\mathcal{T}$回答“什么必须改变”，$\mathcal{P}$回答“什么必须保持”。例如，输入图像中一辆红色汽车停在街道上，指令要求汽车向右行驶，则汽车的位移状态属于 $\mathcal{T}$，汽车身份、颜色、街道背景和相机状态属于 $\mathcal{P}$。这一分解避免将“完成变化”和“保持首帧”混为单一图文相似度问题。

### 3.2.3 E/P/C接口语义

后续评测模块从三个角度消费上述结构化约束：

1. **Execution（E）**：目标变化是否施加到正确主体，并达到指定状态；
2. **Preservation（P）**：主体身份、非目标对象、背景或相机等保持项是否稳定；
3. **Coherence（C）**：变化过程是否连续，是否存在闪烁、跳变、回跳或结构坍塌。

本文不在数据集构建章节展开各指标的工具实现。为防止模型在完全未执行指令时仅凭画面稳定获得高分，评测接口采用Execution门控思想，综合分可简记为：

$$
S=E\cdot\bigl(\lambda P+(1-\lambda)C\bigr),
$$

其中 $\lambda$ 由评测协议统一设置。数据集侧的职责是提供计算E/P/C所需的目标主体、目标变化、保持集合和推荐工具，而不是在本章确定检测器或视觉语言模型的具体实现。

## 3.3 五维组合能力体系

### 3.3.1 维度推导

五个正式维度由初始状态 $C_0$ 中可被单一指令独立修改、且当前单图数据能够稳定构造和检查的状态分量推导得到。

| 初始状态分量 | 目标变化 | 正式维度 | 主要保持项 |
|---|---|---|---|
| $\mathcal{A}_0$ | 颜色、材质、表情或局部状态改变 | Attribute Binding | 主体身份、姿态、背景、相机 |
| $\mathcal{S}_0$ | 单主体执行语义动作 | Action Binding | 主体身份、背景、非目标对象 |
| $\mathcal{S}_0$ | 主体产生方向性位移 | Motion Binding | 主体外观、背景、相机 |
| $\mathcal{B}_0$ | 天气、光照或环境元素动态变化 | Background Dynamics | 前景主体、相机、非目标区域 |
| $\mathcal{K}_0$ | 推近、拉远、平移、俯仰等相机变化 | View Transformation | 场景内容、主体身份、相对布局 |

该划分强调“主要变化发生在哪一类状态变量上”。同一prompt若同时要求人物跑动、天空变暗和镜头推近，将同时修改多个状态分量，不适合作为单维度诊断样本。

### 3.3.2 属性绑定

属性绑定要求目标主体的指定属性从初始值变为目标值。属性可以是颜色、材质、表情、局部外观或可观察状态。有效样本必须满足：初始属性在首帧中可辨认、目标属性描述明确、变化对象唯一或可稳定定位。动作幅度较大的指令不归入本维度。

典型失败包括属性未变化、属性施加到错误主体、全局滤镜代替局部属性编辑，以及属性正确但主体身份发生漂移。

### 3.3.3 动作绑定

动作绑定考察单主体是否执行明确的语义动作，如挥手、点头、坐下或转头。动作强调姿态或部位状态随时间展开，不要求主体在画面中产生显著净位移。若任务的核心目标是主体从一个位置移动到另一个位置，则归入运动绑定。

典型失败包括静态复制、错误主体执行动作、动作不完整、动作部位错误和动作过程不连续。

### 3.3.4 运动绑定

运动绑定考察主体相对于背景或画面坐标的方向性位移，如向左、向右、向上或向前移动。该维度必须区分真实主体运动与相机运动造成的视觉错觉。为减少camera-pan shortcut，运动样本的文本中禁止显式相机运动词，相机状态被加入保持集合。

典型失败包括方向错误、主体未移动、相机整体平移冒充主体运动，以及运动过程中主体外观漂移。

### 3.3.5 背景动态

背景动态要求天气、光照、天空、水面、火焰或其他环境元素发生指定变化，同时前景主体保持稳定。该维度的关键是前景—背景解耦，而非对整幅图像施加统一滤镜。

典型失败包括背景未变化、全局调色冒充局部环境变化、前景被背景变化污染，以及背景变化过程中主体消失。

### 3.3.6 视角变换

视角变换考察相机状态 $\mathcal{K}_0$ 的变化，包括推近、拉远、水平平移、俯仰和环绕等。场景对象的身份与相对结构应保持稳定。该维度与运动绑定互为边界：前者以相机变化为目标，后者以主体位移为目标。

典型失败包括运镜方向错误、运镜类型错误、画面仅进行二维缩放、运镜抖动和场景结构崩塌。

### 3.3.7 维度边界与隔离规则

设样本的目标维度为 $d$，其禁用语义集合为 $B_d$，prompt的语义单元集合为 $W(P)$，则维度隔离要求：

$$
W(P)\cap B_d=\varnothing.
$$

| 目标维度 | 必须表达 | 主要排除内容 |
|---|---|---|
| Attribute | 明确属性或状态变化 | 强动作、主体位移、主动运镜 |
| Action | 单主体动作 | 主体净位移、背景变化、主动运镜 |
| Motion | 主体方向性位移 | pan、zoom、tilt等相机词 |
| Background | 明确环境变化 | 主体动作、主体位移、主动运镜 |
| View | 明确相机或视角变化 | 主体属性变化、主体动作目标 |

黑名单只能发现显式词汇冲突，不能完全解决隐含语义混杂。因此本文在规则检查后增加VLM维度一致性判断，并对最终1500条样本进行分层人工抽检。

## 3.4 样本结构化表示

### 3.4.1 样本定义

每条样本表示为：

$$
x_i=(I_i,P_i,M_i,Q_i),
$$

其中 $I_i$ 为输入图像，$P_i$ 为最终文本指令，$M_i$ 为评测元数据，$Q_i$ 为质量与来源审计信息。$Q_i$ 不参与模型输入，用于筛选、复现和错误追踪。

### 3.4.2 核心元数据

| 字段 | 作用 |
|---|---|
| `question_id` | 样本唯一标识 |
| `dimension` | 五个正式维度之一 |
| `first_frame_path` | 输入首帧路径 |
| `prompt` | 模型接收的最终指令 |
| `target_subjects[]` | 带稳定ID的目标主体 |
| `target_relation` | 目标变化或关系的结构化表示 |
| `preservation_set[]` | 必须保持的对象、区域或属性 |
| `evaluator_tools[]` | 下游建议调度的工具集合 |
| `expected_failure_modes[]` | 预期失败类型 |
| `difficulty` | 重算后的任务难度 |
| `semantic_rarity` | 语义频率层级 |
| `source_type` | 样本来源类别 |

其中，`target_subjects[].noun`、目标变化和保持集合不得为空。自然语言prompt只用于生成模型，评测器不应重新从prompt猜测评测目标。

### 3.4.3 可追溯性

候选样本通过 `recipe_id` 和 `source_sample_id` 追溯到Phase 1的解析结果和TIP-I2V原始样本。正式发布的精简manifest不包含内部审计字段，但构建侧保留recipe、VLM解析、QC结论和图像处理记录，以支持错误定位和版本复现。

### 3.4.4 跨阶段Schema完整性约束

结构化字段是否“存在”不等价于其是否“可用”。早期候选manifest虽然保留了 `target_subjects` 等字段，但全量审计发现3517条记录的主体名词均为空、主体描述退化为 `the subject`，且缺少可直接消费的结构化目标变化。这意味着自然语言prompt可能看似完整，下游却无法确定应检查的主体和变化。

为此，本文将Schema完整性纳入样本定义而非事后可选检查。设结构化完整性函数为：

$$
Q_{schema}(x)=q_{subject}(x)\land q_{target}(x)\land q_{preserve}(x)\land q_{trace}(x),
$$

其中四项分别表示目标主体及noun有效、目标变化非空、保持集合非空以及来源链可追溯。任一项失败的样本不得进入正式集。该约束同时规定评测元数据必须由Phase 1/2结构化结果产生，不能仅从最终自然语言prompt反向猜测。

## 3.5 任务难度的可操作定义

### 3.5.1 难度与质量分离

产物质量分析表明，低清图像、无效fallback prompt和生僻词会增加模型失败概率，但这些因素属于数据缺陷，不应被包装为任务难度。本文首先应用质量准入函数 $Q(x)$：

$$
Q(x)=Q_{text}\land Q_{image}\land Q_{semantic}\land Q_{schema}.
$$

只有文本有效、图像质量达标、图文语义一致且结构化字段完整的样本才进入难度计算。质量不达标的样本被修复或剔除，而不是标记为hard。

### 3.5.2 难度因子

对质量合格样本，定义五个归一化难度因子：

- $c_t$：目标变化复杂度。单一、幅度明确的变化取低值；局部、微妙或多阶段变化取高值；
- $c_s$：主体定位复杂度。由主体数量、目标面积、遮挡程度和可区分性计算；
- $c_b$：场景干扰度。由背景复杂度、前景—背景可分离性和非目标对象数量计算；
- $c_r$：语义长尾度。由Phase 1频率层级和主体Tier共同确定；
- $c_e$：可判定复杂度。单一工具可稳定判断取低值，需要多工具联合或存在工具不确定性取高值。

综合难度为：

$$
D(x)=0.30c_t+0.25c_s+0.20c_b+0.15c_r+0.10c_e.
$$

初始阈值设置为：$D<0.33$为easy，$0.33\le D<0.66$为medium，$D\ge0.66$为hard。正式阈值不直接凭经验冻结，而是在人工标注子集上通过排序一致性标定：由至少两名标注者对样本难度独立排序，选择使自动等级与人工等级加权Kappa最高的阈值。

### 3.5.3 维度专属规则

不同维度对 $c_t$ 的定义不同：

| 维度 | 低复杂度示例 | 高复杂度示例 |
|---|---|---|
| Attribute | 大面积、显著颜色变化 | 局部材质、细微表情变化 |
| Action | 单关节、短动作 | 多肢体协调或多阶段动作 |
| Motion | 单一绝对方向 | 深度方向或易与运镜混淆的位移 |
| Background | 单一环境元素变化 | 前景密集条件下的局部背景变化 |
| View | 单一推近/拉远 | 环绕、复合视角或明显三维揭示 |

## 3.6 正式数据集配额

正式版本采用等维度设计，每个维度300条，共1500条。等量设计可以避免总体分数被候选数量较多的Action或Background维度主导。

每个维度内部采用边际分层而非过度细分的笛卡尔硬配额：

- 难度目标比例：easy 40%、medium 35%、hard 25%；
- 语义频率目标比例：common 70%、rare 30%；
- 主体Tier目标比例：T1/T2/T3/T4约为55%/28%/12%/5%；
- fallback样本原则上不进入正式集，修复并重新通过检查后可重新参与筛选。

当某一交叉桶供给不足时，保持维度总量和质量门槛不变，优先满足难度与common/rare两个边际分布，并在数据卡中报告实际偏差，不使用其他维度样本静默补齐。

## 3.7 本章小结

本章将I2V组合性任务形式化为首帧条件下的选择性状态变化，使用Preserve-Transform双轴描述样本约束，并将正式评测范围限定为五个已经具备候选供给的维度。在此基础上，本文定义了与下游评测器解耦的结构化样本接口，提出了“质量准入优先、任务难度后置”的分级原则，并确定每维300条、总计1500条的正式数据集规模。下一章将说明这些设计如何通过真实数据先验、结构化解析、recipe构建和分层筛选转化为可复现的数据产物。
