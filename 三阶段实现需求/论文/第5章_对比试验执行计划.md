# 第5章可行对比试验执行方案

## 1. 实验目标

本计划针对《Phase2产物质量问题分析报告》中的五类问题开展对比实验：提示词低频词、无效fallback、图像清晰度、图像尺寸、主体分布以及词汇×主体正交诊断。实验目标不是构建新的视频评分器，而是为Phase 2数据处理和正式1500条样本筛选选择最可靠的方案。

论文主实验只使用数据质量指标、人工抽检和一个固定I2V模型的小规模下游探针。完整E/P/C评测、多模型排行榜和自动指标—人类相关性属于合作者的Phase 3工作，不在本计划中展开。

## 2. 通用实验协议

### 2.1 固定划分

已从3517条候选中按维度、旧难度和semantic rarity分层生成：

- 开发集：每维50条，共250条；
- 验证集：每维50条，共250条；
- 正式筛选池：排除开发集和验证集后用于最终1500条筛选。

开发集用于选择阈值和修正流程，验证集只进行一次最终比较。随机种子固定为 `20260712`。

### 2.2 配对实验

同一实验中的所有方案处理完全相同的原始样本。例如，清晰度实验中的Lanczos、Real-ESRGAN和SwinIR均作用于同一批图像。配对设计可以消除样本内容差异，使结果主要反映处理方法差异。

### 2.3 人工评价

每条样本由两名标注者独立评价，争议样本由第三人裁决。二分类结果报告比例、95% bootstrap置信区间和Cohen's Kappa；1—5分量表报告均值、中位数及配对Wilcoxon检验。多方案比较时使用Holm方法校正多重比较。

### 2.4 P0前置修复

当前3517条manifest全部存在目标主体名词和结构化目标变化缺失。因此，在正式抽样和主体实验前必须先修复：

1. 使用修正后的Phase 1字段映射重新生成question plan；
2. 对仍缺失的样本使用VLM联合读取首帧、prompt和旧plan；
3. 回填 `target_subjects`、`target_relation` 和维度复核结果；
4. 低置信度、多主体或维度冲突样本进入人工复核。

该步骤属于数据修复前提，不作为某种质量方法的“提升结果”。

## 3. 实验A：Prompt低频词治理

### 3.1 研究问题

不同词汇治理方法能否降低无贡献的文学化低频词，同时保持目标主体、目标变化和维度语义不变？

### 3.2 样本与方案

使用开发集250条进行方案调试，验证集250条报告最终结果。对同一prompt比较：

| 组别 | 方法 |
|---|---|
| A0 | 当前final prompt，不作修改 |
| A1 | 仅修改polish指令：要求使用常见、具体、可测试的词汇 |
| A2 | 规则检测 + 人工维护的低频修饰词替换表 |
| A3 | 仅对异常prompt进行LLM定向简化 |
| A4 | A2预处理 + A3定向简化 + 结构化目标后检 |

低频检测只约束对任务无贡献的修饰词，不自动替换主体名词、动作核心词和必要属性词。

### 3.3 指标

- 无贡献低频修饰词率；
- prompt无效率：空槽位、重复冠词、无主动变化、长度越界；
- 目标语义保持率：改写前后目标主体和目标变化一致；
- 维度纯度：是否只包含目标维度；
- 人工自然度与明确度：1—5分；
- 单条处理成本和API调用次数。

### 3.4 胜出规则

首先要求目标语义保持率和维度纯度均不低于95%；在满足硬门槛的方案中，选择低频修饰词率最低、人工自然度最高且成本可接受的方案。

### 3.5 当前推荐

优先采用A4。它比全量LLM改写更节省调用，并通过改写后的结构化后检控制语义漂移。RareWordBlocker等需要本地控制解码logits的方案不适合当前在线API流水线。

## 4. 实验B：Fallback修复

### 4.1 样本与方案

对现有117条fallback全部实验，不进行抽样：

| 组别 | 方法 |
|---|---|
| B0 | 旧fallback草稿 |
| B1 | 维度专属确定性模板，关键槽位缺失则拒绝生成 |
| B2 | LLM根据首帧描述、结构化目标和模板重新生成 |
| B3 | B2 + 自动检查 + 人工确认低置信度样本 |

### 4.2 指标

- 有效prompt比例；
- 空槽位与重复冠词比例；
- 维度正确率；
- 与结构化目标一致率；
- 能够修复并重新入库的样本数。

### 4.3 当前推荐

采用B3。旧fallback不得直接进入正式数据集。若关键目标字段仍为空，则宁可剔除，也不以自然语言流畅度替代结构完整性。

## 5. 实验C：图像清晰度增强

### 5.1 样本

从开发集按五个维度、人物/非人物、原始长边和模糊程度分层选取120张图像。另选40张含清晰人脸的子集，专门分析人脸增强造成的身份改变。

### 5.2 对比方案

| 组别 | 方法 |
|---|---|
| C0 | 当前Lanczos放大 |
| C1 | Lanczos + 轻度Unsharp Mask |
| C2 | Real-ESRGAN |
| C3 | SwinIR |
| C4 | Real-ESRGAN + GFPGAN，仅用于人脸风险消融 |

不把扩散式强生成恢复作为主候选，因为它更容易在原图中引入不存在的纹理和语义。

### 5.3 输入级指标

- Laplacian variance和Tenengrad：锐度参考；
- NIQE和BRISQUE：无参考感知质量参考；
- 原图与增强图的DINOv2主体特征相似度；
- 人脸子集身份特征相似度；
- Grounding主体检测成功率；
- 人工清晰度、伪纹理率和身份改变率。

上述客观指标不单独决定胜负。NIQE改善但身份改变的方案判定为失败。

### 5.4 小规模视频探针

从120张中再选择30张，每种方案使用同一个固定I2V模型生成视频，共最多120个主实验视频；C4只在人脸子集单独分析。人工比较视频清晰度、身份保持和明显伪影，不进行多模型排名。

### 5.5 胜出规则

身份改变率不得显著高于C0，主体特征相似度下降不得超过开发集冻结的容差；满足保真门槛后，选择人工清晰度和主体检测成功率最高的方法。

### 5.6 当前推荐

论文实验重点比较C2与C3。GFPGAN只作为风险消融，不默认用于正式集。若两种超分均明显改变身份，最佳方案应是保留C0/C1并剔除极低清候选，而不是强行增强全部图像。

## 6. 实验D：目标比例与尺寸适配

### 6.1 第一阶段：4:3与16:9

从六个原始宽高比分桶中各选10张，共60张。使用同一种保守适配方法分别生成4:3和16:9版本，比较：

- 待评I2V模型能否直接推理；
- 自动二次裁剪或拉伸情况；
- 主体保留率；
- 几何形变；
- 30张子集的生成成功率和边缘伪影率。

如果论文或目标模型明确要求4:3，则4:3作为正式规格，16:9作为当前工程基线；否则根据模型兼容性冻结目标比例。

### 6.2 第二阶段：尺寸适配策略

在第一阶段选定的目标比例下，对120张宽高比分层样本比较：

| 组别 | 方法 |
|---|---|
| D0 | 非等比直接缩放 |
| D1 | Center Crop |
| D2 | Letterbox |
| D3 | Blur或Reflection Padding |
| D4 | Saliency-aware Crop |
| D5 | Outpainting，仅作实验上界 |

### 6.3 指标

- 主体bbox/mask保留比例；
- 目标主体DINOv2相似度；
- 关键点或轮廓几何形变；
- 接缝、黑边和重复纹理比例；
- 图文一致性；
- 人工总体可用率。

### 6.4 当前推荐

最终方案不应是一种方法处理全部图像，而应采用分档混合策略：

- 与目标比例偏差≤4%：等比缩放或极轻微裁剪；
- 偏差4%—30%：Saliency-aware Crop；
- 偏差>30%且裁剪会损失主体：Blur/Reflection Padding；
- Outpainting不进入默认主流程。

## 7. 实验E：主体分布控制

### 7.1 前提

该实验只能在P0目标主体修复完成后进行。主体分层结合TIP-I2V内部频率、COCO/LVIS类别、WordNet和wordfreq，不能只凭单一词表决定。

### 7.2 对比样本集

从质量合格池中构建两个等规模集合，每维100条，共500条：

- E0：维度内自然随机抽样；
- E1：维度内按T1/T2/T3/T4约55%/28%/12%/5%分层抽样。

两组保持难度和common/rare边际分布尽量一致，避免主体Tier与任务难度混杂。

### 7.3 指标

- T1—T4实际分布；
- 主体类别数、Shannon entropy和头部主体占比；
- Grounding检测成功率，仅作为数据可评测性信号；
- 人工代表性评分；
- 五个维度之间的主体分布差异；
- 正式抽样后各层候选供给缺口。

### 7.4 当前推荐

正式1500条采用E1分层抽样，但T4只作为少量鲁棒性探针。若某一Tier供给不足，应在数据卡中报告偏差，不通过替换主体或构造虚假recipe补齐。

## 8. 实验F：词汇×主体正交诊断（可选）

### 8.1 设计

只使用候选池中自然存在的组合，构建：

| 象限 | Prompt词汇 | 主体 |
|---|---|---|
| F-A | Common | T1/T2 |
| F-B | Rare | T1/T2 |
| F-C | Common | T3/T4 |
| F-D | Rare | T3/T4 |

目标每格40条，共160条；若F-D自然供给不足，则报告实际样本数，不使用LLM主动替换主体补齐。

### 8.2 分析

使用同一个固定I2V模型进行小规模探针，人工比较四象限的指令完成率和主体保持率，计算词汇差异、主体差异和交互趋势。该实验只作为附加诊断，不计入正式主评分。

### 8.3 优先级

该实验为可选P2。若时间或计算资源有限，应优先完成A—E，F可放入未来工作。

## 9. 实验G：难度标签标定

### 9.1 方案

在开发集250条上比较：

- G0：现有easy/medium/hard；
- G1：只使用变化复杂度；
- G2：变化复杂度、主体定位、背景干扰、语义长尾和可判定复杂度的加权分数；
- G3：利用人工难度标签标定G2权重或阈值。

### 9.2 指标

- 与人工标签的加权Kappa；
- easy/medium/hard的供给数量；
- 权重扰动后的标签稳定性；
- 借用合作者评测结果检查难度升高时模型表现是否总体下降。

最后一项只验证数据集难度分层的诊断价值，不研究评测指标本身的可靠性。

### 9.3 当前推荐

优先G3；若250条不足以稳定拟合，则采用公开权重的G2，并报告敏感性分析。

## 10. 执行优先级与最小可行版本

### P0：正式数据集必做

1. 结构化目标修复；
2. 实验B：清除无效fallback；
3. 实验A：Prompt治理；
4. 实验G：难度重标；
5. 实验E：主体分层；
6. 每维筛选300条。

### P1：第5章核心对比实验

1. 实验C：Lanczos、Real-ESRGAN、SwinIR；
2. 实验D：4:3/16:9与三类主要适配策略；
3. 组合消融：Baseline、+Text、+Image、+Sampling、Full。

### P2：资源允许时完成

1. GFPGAN人脸风险消融；
2. Outpainting上界；
3. 词汇×主体正交诊断。

如果时间有限，完成P0和P1即可形成完整论文证据链；P2不应影响正式数据集交付。

## 11. 建议的论文结果表

第5章至少报告以下表格：

1. Prompt五方案的低频率、语义保持率、维度纯度和自然度；
2. 117条fallback三方案的修复成功率；
3. 图像增强四方案的清晰度、身份保持和伪纹理率；
4. 尺寸适配方案的主体保留率、形变率和人工可用率；
5. 自然抽样与分层抽样的主体分布、熵和可评测率；
6. 旧难度与新难度的人工一致性；
7. 完整质量流水线的逐步消融结果。

所有未实际运行的方案只能写成“候选方法”或“实验假设”，不能写成已经获得的提升结论。
