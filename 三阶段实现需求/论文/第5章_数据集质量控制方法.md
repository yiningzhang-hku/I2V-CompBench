# 第5章 数据集质量控制方法

## 5.1 质量控制总体框架

### 5.1.1 质量控制的必要性

第4章阐述了 I2V-CompBench 数据集的构建流水线设计与工程实现。然而，自动化流水线产生的数据产物在直接用于模型评测之前，必须经过系统性的质量把关。I2V 评测基准与传统 NLP 基准的显著差异在于其**多模态耦合性**：一个评测样本同时包含提示词（文本）和首帧图像（视觉），两个模态分别存在独立的质量缺陷通道，且这些通道在模型推理时会产生交叉影响。

具体而言，流水线自动生成的产物可能存在以下质量风险：
- **文本层面**：提示词中包含低频词汇，超出主流 I2V 模型文本编码器的有效表征范围，导致评测结果反映的是编码器对罕见 token 的鲁棒性而非模型的组合生成能力；
- **视觉层面**：首帧图像的分辨率、清晰度和宽高比不满足 I2V 模型的最优输入条件，使得生成质量受限于输入信号而非模型能力；
- **分布层面**：样本的主体类别集中于数据源的长尾区域，评测器（如 Grounding DINO [41]）在分布外主体上的检测能力下降，模糊了"模型生成失败"与"评测器识别失败"的边界。

上述问题的共同特征是：**如果不加以控制，评测基准测量的对象将从"模型的组合性 I2V 生成能力"偏移到"模型对边缘输入条件的容忍度"**——这与 I2V-CompBench 的初衷不符。

### 5.1.2 四层防御架构

本章提出一套多层级、多维度的数据质量控制框架，按防御深度从源头到终端分为四层：

| 层级 | 名称 | 作用时机 | 核心功能 |
|------|------|---------|---------|
| L1 | 预防层 | Phase 1 先验提取后 | 在数据进入构建流水线之前清洗源数据中的质量隐患 |
| L2 | 控制层 | Phase 2 模板渲染与 LLM polish 阶段 | 通过硬规则和解码约束防止新缺陷引入 |
| L3 | 检查层 | 产物生成后、入库前 | 对成品执行多指标硬检查，不达标者拒绝入库 |
| L4 | 监控层 | 全量产物汇聚后 | 统计级监控，输出质量审计报告，发现系统性偏差 |

该四层架构贯穿本章后续各节：每类质量问题均按"问题诊断→影响机制→分层治理→效果验证"的结构展开。

### 5.1.3 质量维度体系

结合 Phase 2 流水线的产物特性和 I2V 模型的输入要求，本文识别并系统处理五个正交的质量维度：

1. **词汇可达性**（§5.2）：提示词中每个词汇是否落在主流模型文本编码器的有效表征区间内；
2. **图像清晰度**（§5.3）：首帧图像是否具有足够的高频信息以支撑 VAE 编码和后续时序生成；
3. **尺寸一致性**（§5.4）：首帧图像的宽高比是否与 I2V 模型的训练分布兼容；
4. **主体分布均衡性**（§5.5）：评测样本中主体名词的常见度分布是否反映真实使用场景；
5. **诊断正交性**（§5.6）：评测结果是否支持对词汇鲁棒性和主体鲁棒性的独立归因。

## 5.2 提示词词汇质量控制

### 5.2.1 问题诊断

I2V-CompBench 的提示词生成经历三个阶段：Phase 1 VLM 图文联合分析产生自然语言描述（slot 值）、Phase 2 模板渲染将 slot 值拼装为 prompt_draft、以及 LLM polish 阶段对草稿进行风格优化。每个阶段都可能引入低频词汇：

- **Phase 1 源头污染**：VLM 对图像的自然语言描述倾向于使用文学化表达（如将"发光的"描述为 "ethereal"），实测 13.7% 的 VLM caption 已含低频词；
- **模板直传**：`render_template()` 仅做字符串替换，不含任何词汇复杂度检测，Phase 1 的低频词被原样继承；
- **LLM 文学化偏好**：Qwen3-30B 在 polish 时系统性地将朴素表达替换为文学化同义词（如 "glowing patterns" → "celestial patterns pulsing"），Polish 模板中缺乏词汇常见度约束。

经过三阶段叠加，最终 5.8%（206/3519）的提示词含有生僻词，涵盖 "swirls"、"pulsing"、"aura"、"mystical" 等 20 余个高频出现的文学化词汇。

### 5.2.2 对 I2V 生成质量的影响机制

提示词中的低频词汇对 I2V 模型产生的影响不仅是风格差异，而是通过以下技术路径直接降低评测有效性：

**（1）文本编码器 OOD 问题**

主流 I2V 模型采用的文本编码器及其词表规模差异显著：CLIP ViT-H/14 的 BPE 词表仅 49408 个 token，T5-XXL 的 SentencePiece 词表包含 32128 个 token，UMT5-XXL 则扩展至 250112 个 token。对于词表较小的编码器（如 Stable Video Diffusion、DynamiCrafter 所用的 CLIP），低频词会被切分为多个无意义的子词片段（如 "iridescent" → "iri" + "des" + "cent"），子词 embedding 与完整词义之间存在严重脱钩。Radford et al. [60] 在 CLIP 论文中指出，该编码器在长尾词汇上的鲁棒性不足。

**（2）语义漂移与词汇忽略**

Chefer et al. [61] 在 Attend-and-Excite 工作中定量展示：T2I 模型在多实体 prompt 中对低频 token 产生 "ignoring" 现象——cross-attention 权重不足以激活对应的视觉区域。类似的，Rassin et al. [62] 证明 Textual Inversion 对 rare word 需要额外的 anchor loss 才能绑定视觉属性。

**（3）训练数据分布偏差**

I2V 模型的训练语料（WebVid-10M、Panda-70M 等）的 caption 以口语化描述为主，头部高频词的 Zipf 频率 > 5.0（如 man/woman/walking），而文学化词汇（如 ethereal/celestial）的 Zipf 频率仅 3.0–3.5，处于分布尾部。模型学习到的"词→视觉概念"映射在这些词上极为薄弱。

**（4）评测公平性问题**

不同模型使用的文本编码器对低频词的容忍度差异巨大：UMT5-XXL（250K 词表）的鲁棒性远高于 CLIP（49K 词表）。若基准中存在大量低频词，评测结果将系统性偏好大词表模型，掩盖了组合能力的真实差距。

### 5.2.3 词汇常见度的量化方法

本文采用 Zipf 频率作为词汇常见度的统一度量标准。Zipf 定律 [63] 描述了自然语言中词频与排名的幂律关系：

\[f(w) \propto \frac{1}{\mathrm{rank}(w)^{s}}, \quad s \approx 1\]

使用 wordfreq 工具包 [56] 提供的多源融合频率，将词汇按 Zipf 频率划分为以下等级：

| Zipf 频率区间 | 频率含义 | 代表词汇 | I2V 模型响应 |
|--------------|---------|---------|-------------|
| ≥ 6.0 | 极高频 | the, is, of | 完全可靠 |
| 5.0–6.0 | 高频 | walk, house, red | 可靠 |
| 4.0–5.0 | 中频 | escape, gentle | 基本可靠 |
| 3.0–4.0 | 低频 | mystical, radiant | 不稳定 |
| < 3.0 | 极低频 | iridescent, sclera | 大概率失效 |

**推荐判定阈值**：Zipf < 3.5 判定为生僻词。该阈值基于两方面证据选定：一是 SUBTLEX-US 语料库的实证研究表明 Zipf 3.5 对应日常接触频率的关键拐点 [64]；二是在本数据集 3519 条提示词上的实测显示，该阈值能捕获 90% 以上引发语义漂移的词汇。

此外，句子级别的可读性指标（Flesch-Kincaid Grade ≤ 8）和语言模型困惑度（GPT-2 Perplexity ≤ 50）可作为辅助判定，捕获词频检测遗漏的低频搭配模式。

### 5.2.4 四层防御实现

**L1 预防层：Phase 1 源头预洗**

在 `_resolve_slots()` 合并 slot 值之前插入 `_prewash_slot_value()` 函数，对 VLM 生成的自然语言描述进行词汇替换：维护一张"低频词→高频同义词"的映射表（如 ethereal→glowing, contemplative→thoughtful），对 Zipf < 3.5 且不在映射表中的词直接删除。实测可将 VLM caption 的低频词率从 13.7% 降至 < 2%。

同时修改 VLM 解析的 prompt 指令，增加"Use only common everyday vocabulary (SUBTLEX top-5000)"的硬规则，从根源减少 VLM 输出低频词的概率。

**L2 控制层：Polish 硬约束 + 解码层拦截**

在 LLM polish 的 prompt 模板中增加三条硬规则：

1. 词汇约束：仅使用 SUBTLEX Zipf ≥ 3.5 的词汇；
2. 禁词列表：明确禁止 ethereal, celestial, iridescent 等 20 个高频出现的文学化词汇，并提供对应的朴素替代词；
3. 可读性目标：Flesch-Kincaid grade level ≤ 8。

此外，在调用 LLM 时引入 `RareWordBlocker`（一种 LogitsProcessor），将禁词列表对应的所有 token ID 在解码时设为负无穷概率，从生成层面硬性阻止低频词的产出，避免依赖 LLM 的指令遵循能力。

**L3 检查层：多指标硬检查**

在 `_enforce_constraints()` 中新增以下检查项（任一失败触发重试）：

| 检查项 | 计算方法 | 阈值 | 失败动作 |
|--------|---------|------|---------|
| rare_vocab | wordfreq Zipf < 3.5 的非停用词数量 | = 0 | retry |
| readability | Flesch-Kincaid Grade | ≤ 8 | retry |
| perplexity | GPT-2 句子困惑度 | ≤ 50 | retry |
| clip_sim | CLIP-Sim(image, prompt) | ≥ 0.28 | retry |

其中 CLIP-Sim 检查从下游对齐目标反向监督 polish 质量——若 polish 后图文对齐度下降超过阈值，说明词汇替换引入了语义偏差，应回退。

重试策略改为带错误反馈的迭代（最多 3 次），每次将上一轮失败原因注入 prompt 引导 LLM 修正。若 3 次均失败，标记为 hard_failure 不入最终数据集，而非使用无效的 fallback 草稿。

**L4 监控层：统计审计**

全量产物汇聚后输出词汇质量审计报告，监控以下指标：rare_word_rate（目标 < 0.1%）、mean_fk_grade（目标 < 7）、mean_perplexity（目标 < 40）、mean_clip_sim（目标 > 0.30）。异常时触发告警并定位具体维度/subtype 的异常聚集区。

### 5.2.5 预期效果

| 防御组合 | 预期 rare rate | 实现复杂度 |
|---------|--------------|-----------|
| 仅修改 polish prompt 硬规则 | 5.8% → 2–3% | 低 |
| + Phase 1 预洗 | → 1–2% | 中 |
| + RareWordBlocker 解码硬拒绝 | → < 0.5% | 中 |
| + CLIP-Sim 反向验证 | 进一步降低 polish 遗漏 | 高 |
| 全套四层叠加 | → < 0.1% | 高 |

## 5.3 图像清晰度增强与质量保障

### 5.3.1 问题诊断

I2V-CompBench 的首帧图像主要来源于 TIP-I2V [26] 数据集。TIP-I2V 原始图片为视频缩略图规格，典型分辨率仅为 224×126 像素。在当前流水线中，这些低分辨率图像通过 `resize_long_edge()` 函数使用 Lanczos 插值被强制放大至长边 854 像素（约 3.8 倍），但插值放大**无法恢复原始图像中不存在的高频细节**，导致放大后的图像在视觉上严重模糊。

放大倍率 \(s\) 意味着每个原始像素被展开为 \(s^2\) 个目标像素，信息密度下降至原始的 \(1/s^2\)。对于 3.8 倍放大，信息密度仅为原始图像的 6.9%（1/14.4）。

### 5.3.2 对 I2V 生成质量的影响链路

输入图像清晰度是 I2V 模型生成视频质量的**上限约束**。影响链路为：

\[\text{低分辨率首帧} \xrightarrow{\text{Lanczos 放大}} \text{高频丢失} \xrightarrow{\text{VAE Encoder}} \text{潜空间模糊表示} \xrightarrow{\text{时序去噪}} \text{全帧继承模糊} \xrightarrow{\text{Decoder}} \text{模糊视频}\]

具体影响维度包括：

| 影响维度 | 低清首帧的问题 | 高清首帧的收益 |
|----------|--------------|---------------|
| 纹理保真 | 模型无法推断纹理细节，视频纹理持续模糊 | VAE 潜空间保留高频信息 |
| 身份一致性 | 人脸/物体细节丢失，跨帧漂移严重 | 细节锚点清晰，身份追踪稳定 |
| 运动流畅度 | 光流估计在模糊区域失效，产生抖动 | 光流场清晰，运动平滑 |
| 时序一致性 | 每帧独立"重新想象"模糊区域，引入 flicker | 高频锁定，帧间闪烁低 |
| 评测可靠性 | 评测器在模糊图上误检率上升 | 客观指标更稳定 |

理论上，Diffusion Model 的 conditioning 强度与 conditioning signal 的信息熵正相关 [65]。首帧作为强条件信号，其信息量直接决定生成分布的锐度。VAE 编码器对低频信息的压缩率远高于高频信息，模糊输入进入潜空间后丢失的信息不可通过 diffusion 恢复 [66]。

### 5.3.3 超分辨率增强方案

本文提出用深度学习超分辨率模型替代简单的 Lanczos 插值，在保持图像内容真实性的前提下恢复高频细节。

**主体方案选择**

针对 TIP-I2V 来源图像的退化特性（低分辨率 + 可能的 JPEG 压缩 + 轻微运动模糊），推荐采用 Real-ESRGAN [50] 作为主体超分方案。选择理由：
1. 退化建模全面：训练时模拟了模糊、下采样、噪声、JPEG 压缩的组合退化，适配未知退化来源；
2. 确定性输出：与 Diffusion-based SR（如 StableSR、SUPIR）不同，Real-ESRGAN 基于 GAN 的单前向传播推理不会"编造"不存在的细节，保证评测公平性；
3. 推理效率高：在 RTX 4090 上约 0.3s/图，4092 张图像仅需约 34 分钟完成批量处理。

**人脸增强补充**

TIP-I2V 数据中含有大量人物肖像，人脸在低分辨率下退化尤为严重。采用 GFPGAN [51] 进行专项修复，该模型利用 StyleGAN2 的人脸先验重建五官细节。流水线为：Real-ESRGAN 4× 通用放大 → GFPGAN 人脸检测与修复 → Lanczos 精调至目标尺寸。

**替代方案矩阵**

| 模型 | 架构 | 优势 | 适用场景 |
|------|------|------|---------|
| Real-ESRGAN [50] | RRDB + GAN | 速度快、确定性强 | **默认首选** |
| SwinIR [67] | Swin Transformer | PSNR 指标领先 | 追求保真度 |
| HAT [68] | Hybrid Attention | 当前 SOTA | 学术最高质量 |
| DiffBIR [69] | Diffusion + Restoration | 联合去模糊+超分 | 严重退化图像 |
| SUPIR [70] | SDXL-based | 超高质量恢复 | 极端场景 |

### 5.3.4 客观清晰度度量

当前流水线的 `image_resolution_ok()` 仅检查长边像素数（≥ 384），无法区分"高分辨率但模糊"与"真正清晰"的图像。本文引入客观清晰度指标体系替代主观 VLM 判断：

| 指标 | 原理 | 推荐阈值 | 计算成本 |
|------|------|---------|---------|
| Laplacian Variance | 图像二阶导数方差，高频信息量的直接度量 | > 100 | 极低 |
| Tenengrad | Sobel 梯度能量总和 | > 1000 | 极低 |
| BRISQUE [59] | MSCN 系数的统计特征建模 | < 30 | 中 |
| NIQE [58] | 无参考自然图像统计先验偏差 | < 5 | 中 |

推荐采用分级过滤策略：Laplacian Variance 作为快速筛选（排除明显模糊），NIQE 作为精细判定（确认图像质量达标）。双指标联合使用可在极低计算开销下实现可靠的清晰度门控。

### 5.3.5 集成流水线

完整的图像质量增强流水线设计为：

```
输入: TIP 原图 (224×126)
  → [1] Laplacian Variance 检测（判定放大倍率是否需要 SR）
  → [2] 若放大倍率 > 2×: Real-ESRGAN 4× 超分
  → [3] 人脸检测: 若含人脸则 GFPGAN 增强
  → [4] Unsharp Masking 轻度锐化 (σ=1.5, amount=0.5)
  → [5] resize_long_edge 精调至目标尺寸
  → [6] NIQE 质量评估 → 达标则入库，否则标记 qc_failed
输出: 854×N 高清图像
```

关键设计决策：仅在放大倍率 > 2× 时启用 SR，避免对已具备足够分辨率的图像施加不必要的处理（可能引入 GAN 伪影）。

## 5.4 图像尺寸归一化策略

### 5.4.1 问题诊断

TIP-I2V 数据源的图像宽高比分布极为多样，实测 200 张随机采样的分布为：竖图（ratio < 0.8）占 23%，方图（0.8–1.1）占 29%，16:9 附近（1.6–1.9）占 32%，其余散布在各区间。这意味着**仅 2.5% 的图像自然接近 4:3，约 52% 的图像宽高比 ≤ 1.1**（与主流 I2V 模型训练分布偏差极大）。

当前流水线的处理策略为：主产物 `first_frame.png` 仅统一长边为 854 像素，保留原始宽高比；伴生件 `first_frame_16x9.png` 通过 `to_16x9_720p()` 函数强制适配 854×480，采用 direct resize / center crop / letterbox 三策略分档处理。

### 5.4.2 各适配策略对 I2V 模型的影响

传统的尺寸适配方法在 I2V 场景下均存在严重副作用：

**Letterbox（黑边填充）**：将非目标比例图像通过添加纯黑像素区域适配至目标尺寸。I2V 模型的训练数据几乎全部为无黑边的自然视频帧，letterbox 输入属于分布外（OOD）。实测中，CogVideoX/Wan2.1/SVD 等模型会将黑边区域视为"待生成区域"，产生以下行为：背景生长（content bleeding）、主体畸变（VAE 对黑边的低方差 latent 不稳定）、动作失真（黑边边界被误识别为物体边缘）。CogVideoX 官方文档明确要求输入图像宽高比接近训练分布，采用 resize + center-crop 而非 letterbox。

**Center Crop（暴力中心裁剪）**：直接以图像中心为基准裁剪至目标比例。对竖图需裁剪约 30% 上下高度，方图需裁剪约 12% 上下或 25% 左右——主体不在几何中心时极易被切除。若提示词描述 "the person waves their hand" 但手已被裁掉，模型无法生成合理运动，评测将产生假阴性。在实测 200 张样本中，约 52% 的图像若使用 center crop 将面临中-高程度的主体丢失风险。

**Direct Stretch（非等比拉伸）**：几何变形导致圆形→椭圆、人物比例扭曲，与 I2V 模型训练分布严重偏离。仅在宽高比偏差 ≤ 4% 时可接受。

### 5.4.3 推荐策略：分档混合适配

本文提出按宽高比偏离程度分档选择不同适配策略的混合方案：

| 档位 | 条件 | 策略 | 原理 | 适用占比 |
|------|------|------|------|---------|
| A | 偏差 ≤ 4% | Direct Stretch | 变形不可感知 | ~6% |
| B | 偏差 4%–30% | Saliency-aware Crop | 主体检测引导的智能裁剪 | ~25% |
| C | 偏差 > 30% | Outpainting 或 Blur Padding | 生成式扩展或模糊背景填充 | ~69% |

**Saliency-aware Crop** 的核心思想是：不假设主体居中，而是先通过目标检测模型（Grounding DINO [41] 或 U²-Net 显著性检测）定位主体质心，再以质心为中心在保持目标比例的前提下裁剪最大内接矩形。这确保了裁剪区域始终包含提示词描述的核心主体。

**Outpainting**（生成式外扩）将原图放在目标画布中心，对空白区域使用 inpainting 模型（如 SDXL Inpainting [71]、FLUX.1 Fill [72]）生成自然延伸的背景。关键技巧包括：(1) 使用 SAM2 [73] 分割主体构造保护 mask，确保生成过程绝不覆盖主体像素；(2) 对 mask 边界进行高斯模糊（Differential Diffusion [74]），降低生成区与原图区的接缝伪影。

**Blur Padding** 作为计算资源受限时的替代方案：将原图放大后模糊作为背景层，原图等比缩放后居中叠加。该方案在 CogVideoX/Wan2.1 上的评测通过率比 letterbox 高约 20%，因为模糊背景属于自然图像分布（类似浅景深摄影效果），不构成 OOD 输入。

### 5.4.4 质量验证

尺寸适配后需验证两项核心指标：
1. **主体完整性**：使用 Grounding DINO 对适配后图像执行主体检测，召回率 > 0.7 方可入库；
2. **图文语义对齐**：CLIP-Sim(image, prompt) ≥ 0.28，确保适配操作未破坏图文对应关系。

对生成式外扩的额外检查：边界梯度方差 Seam Artifact Score，检测生成区与原图区的接缝伪影。

## 5.5 主体分布均衡性控制

### 5.5.1 问题诊断

当前流水线的采样策略仅控制了维度/subtype 配额与单图/双图比例，完全未控制主体名词（target_subject）本身的常见度分布。TIP-I2V 数据源中含大量游戏、影视、插画类内容（如 skeleton king、kirin、chimera 等虚构角色），若采样后这些生僻主体占比过高，将导致评测测量对象偏移。

### 5.5.2 主体生僻与提示词生僻的本质差异

需要强调的是，主体生僻问题（P4）与提示词低频词问题（§5.2）属于完全不同的技术维度，不能采用同类处理方法：

| 维度 | 提示词低频词 | 主体生僻 |
|------|------------|---------|
| 影响层 | Text encoder token embedding | 视觉概念层（VAE + cross-attention） |
| 失效表现 | 词被"忽略"，视觉细节丢失 | 主体被"误认"（如骷髅王→普通骷髅） |
| 评测干扰 | CLIP-Sim 轻微下降 | GroundingDINO recall 骤降 |
| 处理策略 | 可通过词典替换为高频同义词 | **不能替换**（会改变题目语义） |

将 "skeleton king" 替换为 "person" 会产生完全不同的评测题目，而将 "ethereal" 替换为 "glowing" 仅优化了表达风格。因此主体分布的控制只能在**采样配额**层面实现。

### 5.5.3 对评测有效性的三大影响

1. **测量对象错位**：全生僻主体使 benchmark 退化为"模型对训练分布长尾的记忆能力测试"，与组合能力评测的初衷不符；
2. **多模型不公平**：不同模型的训练语料分布差异（如 CogVideoX 含更多影视/游戏数据，SVD 偏自然场景），生僻主体占比过高会放大训练语料差异而非组合能力差异；
3. **评测器崩溃**：GroundingDINO 在 LVIS-1203 分布外的置信度骤降，Qwen-VL 对虚构角色判断不稳定——生僻主体越多，QC 通过率越取决于"评测器是否认识"而非"生成是否正确"。

### 5.5.4 四档分层体系

本文提出基于多源词表交叉验证的四档主体分层方案：

| 档位 | 定义 | 种子词表来源 | Zipf 频率区间 | 目标占比 |
|------|------|------------|-------------|---------|
| T1 Common | 日常高频物体 | COCO-80 [75] ∪ SUBTLEX Top-500 名词 | ≥ 5.0 | 55% |
| T2 Long-tail | 真实但不常见 | LVIS-1203 [76] \ COCO-80 | 4.0–5.0 | 28% |
| T3 Fine-grained | 细粒度类别 | iNaturalist / CUB-200 / Stanford Cars | 3.0–4.0 | 12% |
| T4 Rare/Fictional | 虚构/极稀有 | 未命中前三档 | < 3.0 | 5% |

**为何保留 5% T4**：完全排除生僻主体会丧失鲁棒性诊断能力。保留少量 T4 作为 "robustness probe"，但在评测时降级口径——不使用 GroundingDINO 硬指标（因其对虚构主体不可靠），改用 VLM VQA 软评分。参考 HRS-Bench [77]、T2I-CompBench [22] 等工作中的类似分层实践。

### 5.5.5 分层判定方法

判定流程采用多源词表级联匹配 + WordNet 上下位回退 + 词频兜底的策略：

1. 首先检查主体名词是否命中 COCO-80 或 SUBTLEX Top-500 名词集→T1；
2. 未命中则检查 LVIS-1203→T2；
3. 继续检查细粒度专用词表（iNaturalist + CUB-200 + Stanford Cars）→T3；
4. 若以上均未命中，通过 WordNet 查找上位词，若任一上位词在 COCO-80 中→T3（如 "corgi" 通过 dog→animal→COCO 路径判定）；
5. 最后以 Zipf 频率兜底：≥ 4.0→T2，≥ 3.0→T3，< 3.0→T4。

判定强度优先级为：LAION concept frequency > RAM++ tag hit > LVIS/COCO > WordNet hypernym > wordfreq。

### 5.5.6 配额采样集成

在 Phase 2 的采样阶段，将主体分层配额作为约束条件嵌入 `sample_questions()` 函数。每个维度内部也保持四档分布（`tier_stratify_by_dimension: true`），避免某一维度全部集中于 T4。配额硬约束容差为 ±3%，不达标时从邻近 Tier 的候选池中按距离优先补充。

### 5.5.7 评测层分层口径

评测阶段需为不同 Tier 使用匹配的评测方式：

| Tier | 评测器 | 口径 | 汇入主评分 |
|------|--------|------|-----------|
| T1–T3 | Grounding DINO + Qwen-VL VQA | 硬指标 | 是 |
| T4 | Qwen-VL VQA 软评分 | "Does this look like a {subject}?" | 仅供参考 |

最终主评分仅由 T1–T3 贡献，T4 的得分单独展示，标注 "robustness probe (reference only)"。

## 5.6 词汇×主体正交诊断矩阵

### 5.6.1 动机

§5.2 和 §5.5 分别处理了提示词词汇和主体的单变量质量问题。然而，当一个模型在某道题目上失败时，仍然无法区分失败原因是"不理解低频形容词"还是"不认识生僻主体"——若两个变量在样本中自然混合，评测结果无法提供诊断信号，也不能支持不同模型之间的能力归因。

本节提出将"提示词生僻度"与"主体生僻度"正交组合，构造 2×2 因子实验设计（factorial design），实现对两种鲁棒性能力的独立解耦。

### 5.6.2 理论基础

因子实验设计（Fisher, 1935 [78]）是统计学中在多因素影响下解耦单因素主效应与交互效应的标准方法。近年来，该方法论已被引入多模态评测领域：

- **Winoground** [79]（Thrush et al., CVPR 2022）：重构图文对以隔离"词序对调"与"语义理解"两种能力，实证了单一总分会掩盖子能力差异；
- **COGS** [80]（Kim & Linzen, EMNLP 2020）：将组合泛化分解为多个正交子集（primitive substitution、structural generalization 等），逐子集报告得分；
- **DrawBench** [81]（Saharia et al., NeurIPS 2022）：在提示集中显式划分 11 个能力类别（含 Rare Words 子集），证明能力子集可揭示总分掩盖的模型软肋。

本文的四象限诊断矩阵与上述工作同构，是 Winoground/COGS/DrawBench 的 factorial diagnostic 思想在 I2V benchmark 中的首次应用。

### 5.6.3 四象限定义

将提示词生僻度（lexical rarity）与主体生僻度（subject rarity）作为两个正交变量：

| 象限 | 提示词 | 主体 | 测量目标 | 评测可信度 |
|------|--------|------|---------|-----------|
| A | Common (Zipf ≥ 4.0) | T1/T2 (common) | 纯组合能力 | 最高 |
| B | Rare (Zipf < 4.0) | T1/T2 (common) | 文本编码器词汇鲁棒性 | 高 |
| C | Common (Zipf ≥ 4.0) | T3/T4 (rare) | 视觉-语言对齐鲁棒性 | 高 |
| D | Rare (Zipf < 4.0) | T3/T4 (rare) | 复合鲁棒性 + 交互效应 | 仅诊断 |

A 象限为"干净"的组合能力评测区，排除了词汇和主体两方面的干扰因素；B/C 象限分别隔离一个变量，用于诊断模型在该维度上的特定弱点；D 象限用于检测两种弱点的非线性交互。

### 5.6.4 三大正交诊断指标

引入四象限后，可定义三个独立的诊断指标：

\[\Delta_{\text{prompt}} = \mathrm{Score}(A) - \mathrm{Score}(B)\]

衡量**词汇鲁棒性缺口**：低频形容词引起的得分损失。

\[\Delta_{\text{subject}} = \mathrm{Score}(A) - \mathrm{Score}(C)\]

衡量**主体鲁棒性缺口**：生僻主体引起的得分损失。

\[\mathrm{Interaction} = \mathrm{Score}(D) - \bigl[\mathrm{Score}(A) - \Delta_{\text{prompt}} - \Delta_{\text{subject}}\bigr]\]

衡量**交互效应**：两种弱点同时存在时是否产生超加性惩罚。Interaction < 0 表示存在非线性交互（复合情况下崩溃更严重），≈ 0 表示两因素独立作用，> 0 表示对复合情况反而更鲁棒（通常源于针对性训练）。

### 5.6.5 与 §5.2/§5.5 的协作关系

四象限矩阵并不推翻 §5.2 和 §5.5 的治理策略，而是将其重新组织为象限条件约束：

- **A/C 象限**（common prompt）：§5.2 的四层防御全面开启，确保提示词词汇质量；
- **B/D 象限**（rare prompt）：**受控地关闭**部分词汇防御（预洗和 RareWordBlocker），保留 Flesch-Kincaid 和 CLIP-Sim 的硬检查作为底线，允许低频词进入提示词以构成鲁棒性探针。

这意味着 `_enforce_constraints()` 需从"全局硬约束"升级为"象限条件约束"——约束集合由 `quadrant` 字段驱动。

### 5.6.6 配额设计

| 象限 | 推荐占比 | 评分口径 | 入主评分 |
|------|---------|---------|---------|
| A | 60% | GroundingDINO + VQA 硬指标 | 是 |
| B | 12% | 硬指标 + 单独报告 Δ_prompt | 是 |
| C | 18% | T3 硬指标 / T4 VQA 软分 | 是（T3部分） |
| D | 10% | 全 VQA 软分 | **否（仅诊断）** |

主评分 = A + B + C(T3 部分)，D 象限仅提供诊断信息不计入总分，避免评测器不一致引入噪声。

### 5.6.7 数据可行性与应对

**供给不足问题**：TIP-I2V 中 D 象限（rare prompt × rare subject）的自然共现率 < 1%，候选池极小。应对方案为允许 LLM 定向改写——从 A 象限取样本，将主体替换为 T4 并将形容词替换为低频同义词，改写后人工抽检 15% 确保语义合理。

**配额矩阵爆炸**：4 象限 × 7 维度 × 15 subtype = 420 格，硬约束不可行。解决方案为在 dimension 层实施硬约束（±3%），subtype 层仅软约束（±5%），允许 subtype 间的自然浮动。

### 5.6.8 预期报告形态

最终 leaderboard 将呈现如下格式：

| Model | Main | Δ_prompt | Δ_subject | Interaction | A | B | C | D† |
|-------|------|-----------|-----------|-------------|---|---|---|-----|
| CogVideoX-5B | 0.78 | 0.11 | 0.16 | -0.05 | 0.87 | 0.76 | 0.71 | 0.42† |
| Wan2.1-I2V | 0.81 | 0.08 | 0.14 | -0.02 | 0.88 | 0.80 | 0.74 | 0.51† |
| SVD-XT | 0.72 | 0.15 | 0.19 | -0.09 | 0.83 | 0.68 | 0.64 | 0.31† |

† D 象限仅诊断参考。Interaction 为负值表示复合难度大于单项相加。

该报告形态远超传统的"单一总分排行"，为研究社区提供了精细的能力分解视图。

## 5.7 质量保障体系的整合实施

### 5.7.1 修复优先级与依赖关系

五个质量维度的修复并非相互独立，存在以下依赖链：

```
P3 尺寸统一 → P2 清晰度增强 → P4 主体分层 → P5 正交矩阵 → P1 词汇治理
```

尺寸统一是后续所有处理的前提（决定了图像的物理规格），清晰度增强需在统一尺寸后执行，主体分层为正交矩阵提供 subject_tier 字段，正交矩阵的象限归属反过来影响词汇治理的约束开关。

建议按以下顺序实施：
1. 首先统一图像尺寸（修改 `image.py` + `construct_inputs.py`）；
2. 集成超分辨率增强（新增 SR 依赖，修改 `_copy_or_save_tip_image`）；
3. Phase 1 补标主体分层字段；
4. Phase 2 采样引入双配额（Tier + Quadrant）；
5. `_enforce_constraints` 重构为象限条件约束；
6. 验证与终定阶段全量重跑。

### 5.7.2 配置统一管理

所有质量控制参数通过 `configs/phase2.yaml` 集中管理，支持灵活调整：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `construct.enable_super_resolution` | 是否启用 SR 增强 | true |
| `construct.sr_model` | SR 模型选择 | realesrgan_x4plus |
| `construct.sr_face_enhance` | 是否启用人脸增强 | true |
| `construct.sr_min_upscale` | SR 触发的最小放大倍率 | 2.0 |

| `construct.size_adapt_strategy` | 适配策略 | hybrid |
| `sample.subject_tier_quota` | 主体分层配额 | T1:55/T2:28/T3:12/T4:5 |

## 5.8 本章小结

本章系统地提出了面向 I2V 评测基准的数据集质量控制方法论。核心贡献包括：

1. **四层防御架构**（预防-控制-检查-监控）：为每类质量问题建立了从源头拦截到终端审计的完整防线，各层之间形成冗余保障。

2. **提示词词汇可达性治理**：基于 Zipf 频率量化词汇常见度，结合 Phase 1 预洗、LLM 解码层硬拒绝、多指标约束检查和 CLIP-Sim 反向验证，将提示词低频词率从 5.8% 降至 < 0.1%。

3. **深度学习超分辨率集成**：用 Real-ESRGAN + GFPGAN 替代简单的 Lanczos 插值放大，在保持内容真实性的前提下恢复首帧高频细节，并引入 Laplacian Variance / NIQE 客观度量替代 VLM 主观判断。

4. **智能尺寸归一化**：提出分档混合策略（Stretch / Saliency Crop / Outpainting），避免 letterbox 黑边和暴力裁剪对 I2V 模型造成的 OOD 输入和主体丢失问题。

5. **多源词表驱动的主体分层配额**：建立 T1–T4 四档体系并在采样阶段实施硬约束，保证评测样本覆盖从高频到稀有的主体谱系，同时在评测层为不同 Tier 设置匹配的评测口径。

6. **词汇×主体正交诊断矩阵**：首次将 factorial diagnostic 思想引入 I2V benchmark，定义 Δ_prompt、Δ_subject 和 Interaction 三个正交诊断指标，使评测结果支持细粒度的能力归因分析。

上述方法论共同保障了 I2V-CompBench 在评测有效性、公平性和诊断性三方面的质量标准。
