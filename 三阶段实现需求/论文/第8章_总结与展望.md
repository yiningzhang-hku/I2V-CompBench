# 第8章 总结与展望

## 8.1 论文工作总结

本文围绕图像到视频（I2V）生成模型的组合性能力评测这一核心问题，设计并构建了 I2V-CompBench——一个面向组合性评测的 I2V 生成基准数据集。工作涵盖从评测体系设计、数据集构建到质量保障的完整链条，具体完成了以下研究内容：

**评测体系设计**（第3章）：提出了 Preserve-Transform 双轴框架，将 I2V 模型需要满足的约束分解为"变化执行"和"非目标保持"两类正交要求，并进一步细化为 Execution/Preservation/Coherence 三维评分体系。定义了七个评测维度，建立了从初始视觉状态 $C_0$ 到评分函数 $S = F(E, P, C)$ 的完整形式化链路。

**数据集构建方法**（第4章）：设计了先验驱动的三阶段构建流水线。Phase 1 对 TIP-I2V 百万级真实用户数据进行 VLM/LLM 联合分析，提取频率分层、共现矩阵和 forbidden_words 等先验包；Phase 2 以模板+配额机制将先验包转化为评测样本，通过多步骤的提示词定稿流水线和结构化元数据生成约 3500 条高质量样本。

**数据集质量控制方法**（第5章）：提出了四层防御架构（预防-控制-检查-监控），系统性地解决了流水线产物在词汇可达性、图像清晰度、尺寸一致性、主体分布均衡性和诊断正交性五个质量维度上的缺陷。特别地，引入了词汇×主体正交诊断矩阵，使评测结果支持对不同鲁棒性维度的独立归因分析。

**评测方法设计与实验框架**（第6–7章）：给出了 E/P/C 三维评分框架的具体评测工具实现方案和维度-指标映射关系，设计了涵盖多架构、多规模 I2V 模型的实验框架，展示了数据集在维度级、Tier 级和象限级三个层面的诊断能力。

## 8.2 主要贡献与创新点

本文的主要贡献归纳为以下五点：

1. **Preserve-Transform 双轴评测框架**：首次将 I2V 生成任务的评测约束分解为变化执行与非目标保持两个正交维度，突破了传统视频质量评测的单维度局限。基于此框架定义的 E/P/C 三维评分能够区分"不能执行"与"执行但破坏"两类本质不同的失败模式。

2. **七维度组合能力分类体系**：基于 I2V 首帧条件生成的任务特性，定义了 Attribute Binding、Action Binding、Motion Binding、Spatial Composition、Background Dynamics、View Transformation 和 Interaction Reasoning 七个评测维度，实现了对组合能力的细粒度覆盖。每个维度配备维度隔离约束（$W(P) \cap B_d = \emptyset$），保证样本仅考察单一目标能力。

3. **先验驱动的数据集构建方法**：区别于纯人工构造或无约束自动生成，本文提出以真实用户数据的统计先验指导评测样本合成。频率分层（Tier 1–4）、共现矩阵和 forbidden_words 三重先验机制确保样本既反映真实使用分布又满足评测可控性。

4. **多维度数据质量控制方法论**：提出四层防御架构，针对提示词词汇复杂度（Zipf 频率 + LogitsProcessor 解码拦截）、图像清晰度（Real-ESRGAN 超分 + 客观度量）、尺寸适配（Saliency Crop + Outpainting 混合策略）和主体分布（多源词表四档配额）建立了系统性的质量保障体系。

5. **词汇×主体正交诊断矩阵**：将 factorial experimental design 引入 I2V benchmark，通过 2×2 象限设计解耦词汇鲁棒性和主体鲁棒性，定义 $\Delta_{\text{prompt}}$、$\Delta_{\text{subject}}$ 和 Interaction 三个诊断指标，使评测结果超越单一排行，支持细粒度的模型能力归因。

## 8.3 局限性分析

本文工作存在以下局限，需要在未来研究中逐步改进：

**数据来源的单一性**：当前数据集的先验来源仅为 TIP-I2V 数据集。TIP-I2V 虽然规模达百万级，但其内容分布受采集平台的用户群体特征影响，可能对某些应用场景（如医疗影像、工业检测等垂直领域）的覆盖不足。

**维度活跃度限制**：七个设计维度中，Spatial Composition 和 Interaction Reasoning 当前因数据供给不足尚未完全激活。这两个维度的样本构建依赖于多图输入场景，对数据源的要求更高。

**评测器依赖性**：自动评测的可靠性受制于 VLM、Grounding DINO 等评测工具自身的能力边界。当评测对象涉及极端场景（如高度抽象的艺术风格、极端光照条件）时，评测器的判断可能不可靠。

**静态数据集的时效性**：I2V 模型发展迅速，当前数据集的难度设定可能在未来被新模型轻松超越。缺乏动态更新机制可能导致基准"饱和"。

## 8.4 未来工作展望

基于上述局限性和 I2V 领域的发展趋势，未来工作可从以下方向推进：

**维度扩展与多源先验融合**：引入更多数据源（如 Pexels、Shutterstock 等高质量视频平台的用户 prompt 数据），激活 Spatial Composition 和 Interaction Reasoning 维度，并探索新的评测维度（如物理合理性、因果推理、多轮条件生成等）。

**动态难度自适应机制**：设计基于模型能力反馈的难度动态调整策略——当当前样本集的区分度下降时，自动从更高 Tier 或更复杂的组合模式中生成新样本，保持基准的持续挑战性。

**多模态评测器升级**：随着 VLM 能力的持续提升（如 GPT-4V、Gemini Pro 等），评测器的可靠性将显著改善。未来可探索用更强的 VLM 替代当前的 Qwen2.5-VL，或引入人机协同评测模式（AI 初筛 + 人工复核）进一步提升评测一致性。

**跨任务迁移与统一评测**：I2V-CompBench 的 Preserve-Transform 框架本质上适用于所有条件生成任务（如 Image Editing、Video Editing、I2V+Audio 等）。未来可将该框架推广为统一的条件生成评测范式，建立跨任务可比的能力坐标系。

**开放社区协作**：将数据集和评测框架发布为开源项目，鼓励社区贡献新维度的样本、新模型的评测结果和新评测工具的适配，形成持续演进的活跃基准生态。
